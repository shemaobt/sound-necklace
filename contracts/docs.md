# Noridoc: contracts

Path: @/contracts

### Overview

- Zod-4-validated DTOs, mappers, and **the serializer** — the single place where the app's artifacts and exchanged payloads take their exact byte shape: `bead-manifest.json`, `anchoring-return.json`, the report `.md` builder, session state, pipeline imports, and API/bucket payloads.
- The artifact export trio is complete: @/contracts/serialize.ts (the one JSON serializer + export filenames), @/contracts/manifesto.ts and @/contracts/retorno.ts (schema + mapper + export gate each), and @/contracts/relatorio.ts (the `<slug>-mapping-report.md` builder), plus valid/invalid schema fixtures under `fixtures/`. ENG-235 added @/contracts/api.ts and @/contracts/bucket.ts (endpoint + bucket DTOs, since reconciled with the real tripod-api OpenAPI — see below). ENG-234 completed the layer with @/contracts/session-state.ts (versioned full-session autosave DTO + round-trip mappers) and @/contracts/imports.ts (the two pipeline import mappers).
- Exists as its own frozen layer because byte-identity of artifacts is the product's core promise to the downstream pipeline. Mappers are 1:1 ports of the reference's export functions in @/docs/reference/index.html.

### How it fits into the larger codebase

- Imports **only** @/domain (plus `zod`); enforced by @/.dependency-cruiser.cjs. Consumed by @/adapters and the `ui/` wiring layer.
- The golden harness (@/tests/golden) byte-diffs this layer's output against goldens generated from the reference @/docs/reference/index.html. Since ENG-227 the harness replayers export through the **production** mappers (`buildManifesto` + `serializeArtifact` in @/tests/golden/registry.ts) — the byte guarantee covers this layer directly, not a local mirror.
- Mappers take the domain's `SessionState` (@/domain/state.ts) as their sole input; export gates are exposed as predicates/status objects for the caller (ui) to render, mirroring the reference's download handlers.
- Custody rule downstream (PRD §10.5 in @/docs/PRD-colar-de-sons-v2.md): every system that stores or transports the artifacts (API, dashboard download, pipeline fetch) treats them as **opaque bytes**. `SessionStore.complete()` in @/adapters receives artifact bytes it must never re-serialize. The bytes produced here are the bytes o Compilador receives.

### Core Implementation

- **Serializer** (@/contracts/serialize.ts): `serializeArtifact` is `JSON.stringify(value, null, 2)` — raw UTF-8 (non-ASCII never `\u`-escaped), no BOM, **no trailing newline**; a 1:1 port of the reference's `download()`. No other JSON serialization path may exist in the app. `manifestoFilename`/`retornoFilename` carry the reference's `"colar"` fallback — **filename-only**: the `story_slug` inside the retorno JSON is the raw slug. The report has its own divergent fallbacks (see Report below).
- **Manifesto** (@/contracts/manifesto.ts): `ManifestoSchema` is a `z.strictObject` (`manifest_id` matches `fnv1a32:` + 8 hex). `buildManifesto` mirrors the reference's `buildManifest()` exactly — literal key order is part of byte-identity, and each bead is re-projected into a **fresh** `{index, startTime, endTime}` object in that order. `canExportManifesto` exposes the reference gate: without a bead grid, export is a **silent no-op**.
- **Retorno** (@/contracts/retorno.ts): `RetornoSchema` is strict throughout; `scene_kind` is a runtime-derived enum over the generated `SCENE_KINDS` list from @/domain (the frozen domain deliberately has no union type for kinds); confidence is the `high`/`medium`/`low` enum (English since ENG-356; the legacy PT-BR values are rejected, never coerced); the outer `scenes` array holds **exactly one** scene (the whole story). `buildRetorno` mirrors `buildReturn()`: only `locked && span` items export; `scene_id` is `S#` sequential by list order (reassigned at export, distinct from the stable `part_id`); `scene_kind`/`scene_kind_confidence` use the reference's `||null` coercion; `tag_state` is emitted directly because domain's `TagState` is never falsy — the reference's `||"pending"` would be a dead branch, and bytes are identical without it; propositions come out in global creation order (not grouped by scene); flags are emitted **independent of locked**, so a reopened flagged frase yields a flag with no matching proposition.
- `retornoExportStatus` mirrors the `dlReturn` gate: `canExport` = the whole necklace is confirmed; `semFim` counts unlocked frases with a span or non-blank statement (non-blocking warning).
- **Report** (@/contracts/relatorio.ts): `buildMapReport` is a port of the reference's `buildMapReportMd()` — a fixed Markdown skeleton with load-bearing bytes (em dash U+2014 in the title and before "beads", en dash U+2013 between beads, middle dots U+00B7 in the context line). Level 1 walks `L1_Q`, Level 2 walks `L2_Q` over every locked part (incl. `none_fit`), Level 3 walks `L3_Q` over the phrases of each productive scene. `sceneNum` is the index within `lockedParts`, so `S#` has **gaps** when none_fit scenes precede productive ones; the final empty push guarantees exactly ONE trailing `\n`. Missing cells emit `_(no answer)_`.
- **The report is English** (ENG-356, policy ENG-326) — a deliberate, reviewed divergence from the reference, which emits the same skeleton in PT-BR. Question text comes from `q_en` in @/domain/mapeamento-scripts (`field` was already English; `note` is never emitted, so it stays PT-BR only). The golden `.md` is still _derived_ from the reference: @/tests/golden/generate.mjs applies the hand-written `REPORT_PT_TO_EN` table and throws on any surviving PT-BR marker — see @/tests/golden/README.md.
- **Voice never enters the artifact** (ENG-356): the cell is text-only — the confirmed English text, else `_(no answer)_`. Recordings stay in the bucket and in the session `meta.voice` as provenance and remain playable in on-screen review; `buildMapReport` no longer takes a voice-path set (the old `fixtures/relatorio/` PRD-derived fixture was removed with it).
- **Three distinct slug fallbacks** — all faithful ports, do not "unify": the report FILENAME (`relatorioFilename`) falls back to `"historia"` (NO accent, reference nav L1152); the JSON artifact filenames fall back to `"colar"` (@/contracts/serialize.ts); the report TITLE falls back to `"história"` (WITH accent).
- Schema fixtures live under `fixtures/` — a valid artifact plus invalid variants (missing key, extra key, wrong type, wrong enum) per schema, exercised by the colocated tests.
- **API / bucket** (@/contracts/api.ts, @/contracts/bucket.ts): the endpoint DTOs the SPA consumes, **reconciled with the real tripod-api OpenAPI** during ENG-247 (a filtered production snapshot lives at @/contracts/openapi.json with generated types in @/contracts/generated/tripod.d.ts, refreshed by @/scripts/api-snapshot.mjs). Auth mirrors the house wire: email login returning the `AuthResponse` envelope (user + rotating token pair), app roles via `my-roles`. Sessions/artifacts/resources mirror the wire verified in ENG-267: `CompleteSessionRequest` and `ResourceRef` were **removed** (they never existed on the wire — `POST /complete` has no body; artifacts travel as raw multipart bytes), replaced by `ArtifactResponse`/`ArtifactUploadResponse` (the multipart upload receipt with per-artifact checksums) and `ResourceSummary`/`ResourceListResponse`/`ResourceUrlResponse` (voice-answer listing + the short-lived signed GET URL for the second, Bearer-less hop). Two custody invariants matter: (1) **artifact payloads are opaque** — `OpaqueArtifactSchema` is `z.string()` and this module contains no `JSON.parse`/`JSON.stringify` call (asserted by a source-grep test), so bytes round-trip untouched (§10.5); (2) the **acousteme envelope** carries the tokenizer grid that resolves granularity — `codebook_version` (a string, confirmed by the real OpenAPI), `hop_sec` + `granularity_frames {small, medium, large}` (§6.1/§15.2 O8), from which the `GranularityResolver` (@/adapters/granularity) derives `beadSec = frames[level] × hop_sec`. The autosave body is typed as a `z.looseObject({schema_version})` here because the real full-state schema is ENG-234's `session-state.ts`; adapters import only the interface types so the swap stays contained. `fixtures/api/bucket-list.json` is the adapter-facing bucket listing (three audios: with/without acousteme, consent varied) — PRD-derived, documented in `fixtures/api/README.md`.
- **Session state** (@/contracts/session-state.ts): the v2-new autosave DTO (PRD §7.3) that lets a session resume from any machine — **no reference 1:1** since the v1 prototype never persisted. `SessionStateDtoSchema` is a `z.strictObject` (bad enum/type/`schema_version`/extra key all reject) that serializes **every** domain `SessionState` field (@/domain/state.ts) including the full grid `beads[]`, plus a separate `SessionMeta` for the v2 fields that don't live in the domain: `granularityLevel` (§8.1, reusing `GranularityLevelSchema` from @/contracts/bucket.ts), `bucketAudioId` (§7.4), `pipelineConsent` (§12/O6), and `voice[]` resource paths (§10.4/O5). `toSessionDto(state, meta)`/`fromSessionDto(dto)` are the round-trip pair. Transient reference state is deliberately NOT persisted — the `warnedEmptyScene` module var (see @/domain/phrases.ts) and the map-screen scaffold (`mapStep`/`mapN*i`) are reconstructed on reopen. `whole.id` is stored as a plain `string` (not the domain's literal `'S1'`) because a delivery import can overwrite it with an external `scene_id`; `fromSessionDto` casts it back (`as Whole['id']`).
- **Pipeline imports** (@/contracts/imports.ts): 1:1 ports of the reference's two interop handlers (@/docs/reference/index.html). `applyDelivery(session, dto)` maps a delivery into **UNLOCKED** proposals — spans from `proposed_span` else null, tag/kind prefills preserved, `PT#`/`P#` index fallbacks, `whole.id` overwritten only when `scene_id` is present, and a **non-blocking** `manifestMismatch` flag (true only when `manifest_id` exists and differs from the session's). `applyReturn(session, dto)` maps a return into **everything LOCKED** — spans from `confirmed_span` (schema-required), `partsConfirmed` set when parts exist and never unset, NEEDS_REVIEW flags reapplied by `prop_id`, cursor moved to `frases`, and **no** manifest check (mirrors the reference). Both return a discriminated `ImportOutcome` and reject with `{ok:false, reason:'no-grid'}` when the grid is absent (`totalBeads` 0). PT-BR copy is exposed as `DELIVERY_NO_GRID_MSG`/`RETURN_NO_GRID_MSG`/`MANIFEST_MISMATCH_MSG` constants for the imports UI (ENG-248) to reuse.
- Zod 4 idioms are mandatory: import from the root `"zod"` only (`zod/v3`/`zod/v4` subpaths are lint-banned in @/eslint.config.js because they mix eras silently); `z.strictObject`, `.extend()` not `.merge()`, two-arg `z.record()`.

### Things to Know

- **FROZEN LAYER / contract-critical:** changes require the golden harness green AND explicit human approval in the PR. Loop agents stop and escalate; never merge autonomously.
- Coverage gate ≥ 90% (glob thresholds in @/vitest.config.ts), and every schema must have both a **valid and an invalid fixture** test.
- The depcruise rule `contracts-so-domain` has **no test exemption**: `contracts/*.test.ts` cannot import from `tests/` — fixtures are read with `node:fs` instead (Node builtins are allowed in tests).
- `scene_kind` stays `string | null` at the type level (the enum is runtime-derived, so Zod's inference widens to `string`) — the 27 kinds are enforced ONLY by `RetornoSchema` at validation time. The export wiring (station issues) must `safeParse` the mapper output before serializing, and always serialize the ORIGINAL mapper literal, never the parse result (Zod reconstructs objects in schema-shape order, which would break byte identity).
- **Strict vs. lenient is intentional:** the import DTO schemas (@/contracts/imports.ts) use lenient `z.object` (ignore unknown keys, faithful to the reference which reads only specific fields), UNLIKE @/contracts/session-state.ts and the artifact schemas which use `z.strictObject`. So an import-fixture invalidity is **semantic** (bad confidence enum, non-int bead, missing `confirmed_span`), never "extra key".
- The API/bucket schemas are no longer provisional: they mirror the published tripod-api OpenAPI (snapshot + generated types under @/contracts/generated, closing O9 from PRD §15.2). Lessons baked in: never model a request the wire doesn't have (the removed `CompleteSessionRequest`), and never be stricter than the wire on a response (`duration_sec` is nullable because the API validates on ingestion, not on listing). Any further drift still goes through the frozen-layer process.
- Byte-identity is fragile by nature: key order, number formatting, and newline discipline all matter. Node and Chromium are both V8, so `JSON.stringify` number formatting matches across the golden-harness boundary — do not introduce any other serialization path.

## Nomes de arquivo e versões do DTO (ENG-357/ENG-359)

- **Filenames em inglês** (@/contracts/serialize.ts, @/contracts/relatorio.ts): `<slug>-bead-manifest.json`, `<slug>-anchoring-return.json`, `<slug>-mapping-report.md`, com fallback único `story` para slug vazio. O `kind` da API (`manifest`/`anchoring`/`report`) continua sendo o **handle do contrato** — o nome do arquivo é rótulo de download e chave de armazenamento. ⚠️ `tripod-api` e Compilador ainda esperam os nomes PT-BR (ENG-358); `contracts/openapi.json` é gerado do backend e por isso ainda descreve o contrato antigo — não editar à mão.
- **DTO de sessão versionado** (@/contracts/session-state.ts): `schema_version` 2 é a forma corrente (inglesa); 1 é a forma PT-BR anterior à ENG-356. `fromSessionDto` recebe `unknown` e é o **único ponto de validação da borda de leitura** — o adapter entrega o corpo cru (`http.ts`: "validado a fundo pela camada de estado"). Aceita v1 e promove para v2; `SessionStateDtoSchema` (v2) **recusa** v1, porque migrar é passo explícito e não tolerância de schema. Reabrir e salvar promove a sessão.
- Buraco conhecido: sessões gravadas na janela de um commit entre a ENG-356 e a ENG-357 carregam a forma v2 carimbada como v1 e reprovam nas duas validações — deliberadamente, já que falhar alto é melhor que reidratar meio estado.
