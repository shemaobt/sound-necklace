# Noridoc: domain

Path: @/domain

### Overview

- The **frozen core**: pure TypeScript, zero imports (no other layer, no npm packages), a 1:1 behavioral port of the v1 reference at @/docs/reference/index.html.
- Modules land in dependency order via the E1 issues, all re-exported through the @/domain/index.ts barrel (which also keeps the `DOMAIN_LAYER` marker). Landed: ENG-214 — bead grid math (@/domain/grid.ts), FNV-1a `manifest_id` hash (@/domain/hash.ts), lowest-free ID allocators (@/domain/ids.ts); ENG-216 — session state (@/domain/state.ts), frontier/active anchor (@/domain/frontier.ts), the bead-click selection model (@/domain/selection.ts), and whole-story + sequential scene cutting (@/domain/scenes.ts); ENG-219 — the generated Ruth scene-kind ontology (@/domain/scene-kinds.ts), the triagem picker mutations + selectors (@/domain/triagem.ts), the coverage derivation (@/domain/coverage.ts), and the mode gates (@/domain/gates.ts); ENG-223 — the phrase lifecycle (@/domain/phrases.ts) and the seam/border-crossing logic (@/domain/seam.ts); ENG-226 — the verbatim mapeamento question scripts (@/domain/mapeamento-scripts.ts) and the lazy answer store (@/domain/mapping.ts). With ENG-226 the E1 domain chain is complete.
- Exists as its own layer so the entire rule set compiles and tests with no framework/IO, making byte-identical artifact reproduction provable by the golden harness.

### How it fits into the larger codebase

- Innermost layer of `ui → adapters → contracts → domain`. The zero-import rule is mechanical, not conventional: @/.dependency-cruiser.cjs errors on any import of another layer **and** on any npm package from production code (tests may import vitest).
- Consumed by: @/contracts (DTO mappers over domain types), @/adapters, `ui/organisms` and `ui/state` (domain types/state via props/hooks — see @/ui/docs.md).
- The golden harness (@/tests/golden) replays scripted decision sets through `domain/` + `contracts/` and byte-diffs the artifacts against goldens generated from the reference — proving `domain ≡ reference` on every PR. This is why the layer is frozen: its behavior _is_ the product contract. The harness replayers in @/tests/golden/registry.ts import through the @/domain/index.ts barrel (grid/hash for the manifest cases; `createSession` and the scene/triagem/phrase/mapping reducers for `replaySessionSteps`).
- Effects are values, never side effects: reducers return new state (plus a `PlayAction` in @/domain/selection.ts and the `BorderOffer` in @/domain/seam.ts); the interpreters — audio playback, border-offer dialogs — live in @/adapters and the UI wiring layers, and UI organisms consume state via props/hooks only.

### Core Implementation

- @/domain/grid.ts — `buildBeads` (grid = `floor(dur/beadSec + 1e-9)` beads plus a partial bead when a remainder > 1e-9 exists; every `startTime`/`endTime` is coerced through `+(x).toFixed(6)` and `endTime` is clamped to the duration), `beadAtTime` (time → clamped bead index, same epsilon), `spanDur` (span duration from bead times; throws on out-of-grid indices — a `noUncheckedIndexedAccess`-driven guard).
- @/domain/hash.ts — `PcmLike` (the minimal AudioBuffer-shaped interface: channels, sampleRate, `getChannelData`) + `hashPCM`, the FNV-1a 32-bit `manifest_id` (`fnv1a32:xxxxxxxx`). Byte-mix order is contract: channel count, the 2 low bytes of sampleRate, sample count as 4 bytes LE, strided int16-quantized samples from channel 0 (stride = `max(1, floor(N/100000))`), then `beadSec*1000` as 3 bytes LE.
- @/domain/ids.ts — `nextPartId` (`PT#`) and `nextPid` (`P#`): lowest free number, scanning ALL entries — locked or not, because pending slots occupy their ID.
- @/domain/state.ts — the COMPLETE `SessionState` shape (whole/parts/frases/current/activeSceneId/mapping/selection/pendingStart/mode/review plus the audio facts: duration, beadSec, bead grid, `manifest_id`, filename, slug) and `createSession`, mirroring the reference's `state` object (reference L367–379) and the reset done by `segment()` (L454–462). Later issues only ADD functions over this shape — they never edit the state module. UI/playback fields of the reference (`color`, `beadSize`, `paused`, download flags, map-step scaffolding, `audioBuffer`) are deliberately excluded: they belong to @/ui and @/adapters.
- @/domain/frontier.ts — `frontier(state, layer)`, the COMPLETE frontier of both layers, and `activeAnchor` (the current unlocked item, if any, with its frontier as anchored start). On the phrase layer with a productive scene in focus it runs the active-scene branch first: last locked phrase end in that scene + 1; with no phrase in the scene, back-reach to the START of the previous locked neighbor (via `prevNeighbor` from @/domain/seam.ts), else the scene's own start — and this branch returns BEFORE the clamp. Only when no productive scene is in focus does it fall to the generic branch (max locked end + 1, capped at the last bead; the phrase layer floors at `whole.span.s`). Housing the full branch here (rather than only in @/domain/phrases.ts) is what makes `activeAnchor().start` — and therefore the `clickBead` clamp in @/domain/selection.ts — honor back-reach; the earlier split left clicks floored at 0 before the first phrase.
- @/domain/selection.ts — `clickBead`, the DEFINING click model (owner decision, replaces the reference's two-click/nudge model; @/docs/segmentation-rules.md rule 1). The segment start is FIXED at the frontier — the user only ever sets the END. Returns `{state, play}` with a discriminated `PlayAction`: `transport` (no active anchor → play the tapped bead, state untouched); `listen` (clicked the start/frontier → the UI plays from there to the parent end; selection untouched); `set-end` (clicked beyond → `selection = {frontier, bead}`, `pendingStart=null`; the UI decides stop-if-the-playhead-passed-else-continue). The play/stop/continue decision depends on the runtime playhead and lives in the UI (@/ui/pages/cut/cutting.ts `playClick`), not here.
- @/domain/scenes.ts — whole-story confirm and the sequential scene-cutting reducers (`addPart`/`primePart`/`confirmPart`/`confirmParts`, reference L685–767 and `enterLayer('parts')` L930–935). All reopen (whole L677–680 and scene L726–731) was removed in ENG-342 — a locked scene is adjusted by dragging its boundary (`dragSceneBoundary`, @/domain/seam.ts) or removed with `removePart` (mirror of `removeFrase`; not in the reference, added to give scenes the phrase's post-hoc removal). Both `removePart` and `removeFrase` stay pure/reference-faithful (the golden replays them directly); the owner-approved "next item absorbs the freed gap" (#3) is a SEPARATE pure step — `absorbNextScene` / `absorbNextFrase` — that the station composes AFTER the remove (like the post-drag re-anchor), so it lives outside the golden's byte-identity scope. The story is un-confirmed via the Cortar back action (`setMode('escuta')` + `whole.confirmed=false`), not a reducer here. Validation failures return a typed `Result` (`{ok:false, error:{code, message}}`) whose PT-BR messages are contractual copy exported as `SCENE_ERROR_COPY` — the UI must reuse them, never rephrase. `confirmParts` prunes unlocked slots and transitions `mode` to `triagem` directly (the mode gates proper live in @/domain/gates.ts).
- @/domain/scene-kinds.ts — the generated 27-kind Ruth ontology (`SCENE_KINDS`, tier + verbatim order from `_spec/scene-kind-palette.json`, pin 5314907 — never hand-edit). Each kind's English `value` IS the Compilador contract and is exported as-is; `SK_PT` holds PT-BR display labels (display-only, never stored) and `skEnShort`/`skShort` produce the short labels the report `.md` bytes depend on (ENG-233). `T_TARGET` is the per-tier coverage target (ALTA=1, comum=3).
- @/domain/triagem.ts — the picker: mutations `tagScene` (sets `tag_state='tagged'` + `scene_kind` + confidence) and `markNoneFit` (`tag_state='none_fit'`, nulls kind/confidence), plus selectors `lockedParts` (locked && span) and `productiveScenes` (locked && span && tagged, in parts order). Pure immutable maps keyed by `part_id`; no throw.
- @/domain/coverage.ts — `computeCoverage`, a pure port of the reference's `renderCoverage` calc with its DOM side effects and its `updateModeLocks`/`updateTriagemDone` calls stripped (those are now pure in @/domain/gates.ts). Firm = alta|média, hesitant = baixa; per-kind status covered/partial/open; `candidateAbsence` = tier ALTA && firm===0 (the reference's open-rara predicate — ignores hesitant); `allNoneFit` lockout = every locked scene triaged && zero productive. Also exposes additive aggregate derivations (total, rare totals, noneFit, productive, triaged) for the ENG-225 coverage drawer.
- @/domain/gates.ts — the mode gates + triagem hard gate as pure functions: `triagemDone` (enabled iff every locked scene is non-pending && ≥1 productive; its PT-BR helper copy is contract-level, verbatim incl. curly quotes/em dash), `modeLocks` (escuta always; triagem⇔`partsConfirmed`; segmentacao⇔productive>0; mapeamento⇔productive>0 && ≥1 locked phrase with span), `resolveMode` (the setMode redirect — segmentacao/mapeamento with zero productive fall back to triagem; it checks ONLY productive, not phrases), and `setMode` (applies `resolveMode`, always drops `review`, and sets `partsConfirmed=true` when entering segmentacao with locked parts — the reference L1005 side effect).
- @/domain/seam.ts — the seam between scenes, all pure data: `prevNeighbor`/`nextNeighbor` (nearest LOCKED neighbors — none_fit counts; they return `AnchoredPart`, span proven in the type), `sceneHasFrases`, `classifyBorderMove` (the border-crossing offer as a `BorderOffer` value — kinds `simple`/`escalation`/`two-productive`, threshold `max(3, round(0.25·span))`, `consumed` = the selection swallows the whole neighbor, `canMove`, contractual PT-BR copy in `BORDER_COPY`; throws on a productive scene without span/kind — caller-invariant violation, not flow), `slideSeam` (the scene grows, the immediate locked neighbor shrinks; immutable, untouched parts keep identity), and the segmentação render window `windowMargin`/`sceneWindow` (`max(3, round(2/beadSec))`).
- @/domain/phrases.ts — the phrase lifecycle over the productive-scene view of @/domain/triagem.ts: `activeScene`/`sceneIndexOf` (focus resolves by `activeSceneId`, else first productive), `phraseFrontier` (inside the active scene: last locked phrase end + 1; the FIRST phrase back-reaches to the START of the previous locked neighbor, else the scene start), `addFrase` (lowest free `P#` via @/domain/ids.ts; no-op while an anchor is active), `confirmFrase` (guards in the reference's exact order, returning a discriminated `locked`/`border`/`noop`/`error` union with contractual copy in `FRASE_ERROR_COPY`; a border crossing returns the `BorderOffer` — the UI decides), `moveBorder` (slide the seam then lock), `reanchorFrase`, `removeFrase`, `dragPhraseBoundary` (drag a locked phrase's edge — grow it, shrink the immediate neighbor only when they touch; ENG-342 replaced the old reopen/⚑), the navigation entries `enterScene`/`enterFrasesLayer`/`enterSegmentacao`, and `confirmFrasesDone` (empty-scene warning once PER SCENE; advances to the next productive scene or asks for mapeamento via `setMode` from @/domain/gates.ts).
- @/domain/mapeamento-scripts.ts — the verbatim Mapeamento question scripts `L1_Q`/`L2_Q`/`L3_Q` (reference L1030–1056). The strings ARE the byte-exact contract the ENG-233 report `.md` serializes — including load-bearing unicode (curly quotes U+201C/U+201D in the `tempo` note, em dash U+2014 inside the `funcao` question and `ausencia` note) and structural quirks: L2 `descrever` has an EMPTY `field` (not absent), L3 items have NO `field` property at all.
- @/domain/mapping.ts — the answer store over the scripts: `ensureMapping` (lazy create/extend of `state.mapping` — seeds `""` per question with an `== null` check; L2 buckets for ALL locked scenes via `lockedParts` of @/domain/triagem.ts, none_fit included; L3 buckets for every locked phrase with span + `part_link`, even in a non-productive scene; never deletes a bucket), `setAnswer` (direct assignment, the golden driver's semantics; throws on a missing L2/L3 bucket), `productiveFrases` (scene-major order over productive scenes), `questionSequence` (the flat conversation sequence of PRD §8.7: all L1 → L2 per locked scene in order → L3 per productive-scene phrase), and `voiceAnswerPath` (`respostas/level{1,2,3}/…/<k>.webm`, PRD §10.4 / decision O5).
- @/domain/seam.ts — the seam between scenes, all pure data: `activeScene` (the productive scene in focus: resolves by `activeSceneId`, else the first productive from @/domain/triagem.ts — it lives HERE, not in @/domain/phrases.ts, so @/domain/frontier.ts can import it without a cycle), `prevNeighbor`/`nextNeighbor` (nearest LOCKED neighbors — none_fit counts; they return `AnchoredPart`, span proven in the type), `sceneHasFrases`, `classifyBorderMove` (the border-crossing offer as a `BorderOffer` value — kinds `simple`/`escalation`/`two-productive`, threshold `max(3, round(0.25·span))`, `consumed` = the selection swallows the whole neighbor, `canMove`, contractual PT-BR copy in `BORDER_COPY`; throws on a productive scene without span/kind — caller-invariant violation, not flow), `slideSeam` (the scene grows, the immediate locked neighbor shrinks; immutable, untouched parts keep identity), and the segmentação render window `windowMargin`/`sceneWindow` (`max(3, round(2/beadSec))`).
- @/domain/phrases.ts — the phrase lifecycle over the productive-scene view of @/domain/triagem.ts: `sceneIndexOf`, plus `activeScene` re-exported from @/domain/seam.ts and `phraseFrontier` as a pure alias of `frontier(state,'frases')` in @/domain/frontier.ts (both kept so the @/domain/index.ts public API stayed stable when the logic moved), `addFrase` (lowest free `P#` via @/domain/ids.ts; no-op while an anchor is active), `confirmFrase` (guards in the reference's exact order, returning a discriminated `locked`/`border`/`noop`/`error` union with contractual copy in `FRASE_ERROR_COPY`; a border crossing returns the `BorderOffer` — the UI decides), `moveBorder` (slide the seam then lock), `reanchorFrase`, `removeFrase`, `dragPhraseBoundary` (drag a locked phrase's edge — grow it, shrink the immediate neighbor only when they touch; ENG-342 replaced the old reopen/⚑), the navigation entries `enterScene`/`enterFrasesLayer`/`enterSegmentacao`, and `confirmFrasesDone` (empty-scene warning once PER SCENE; advances to the next productive scene or asks for mapeamento via `setMode` from @/domain/gates.ts).
- Each module carries colocated tests (`*.test.ts`). Hash tests pin known vectors computed by an independent Python oracle (the emulation script is quoted in @/domain/hash.test.ts) plus vectors derived from the committed goldens in @/tests/golden/expected, so `domain ≡ reference` holds both by construction and by byte-diff.
- Non-negotiable facts this layer implements exactly (details in @/docs/PRD-colar-de-sons-v2.md §10–§11 and @/CLAUDE.md):
  - Bead indices are the universal coordinate; `S#` sequence IDs are reassigned only on export.
  - Frontier: sequential locking (@/domain/scenes.ts, @/domain/phrases.ts); post-lock adjustment is dragging the END of a scene/phrase (`dragSceneBoundary`/`dragPhraseBoundary`, ENG-342 — now IDENTICAL and **Pac-Man/tiled** for both, owner decision: the NEXT segment follows the boundary in both directions (its start becomes `newEnd+1`), so shrinking never opens a gap — the next grows to fill — and growing pushes the next; the last segment (no next) grows/shrinks freely to the story/scene end). Only the END drags — the start is the seam, moved only by dragging the previous segment's end. See @/docs/segmentation-rules.md for the full rule set. The drag reducers are pure span editors; the station composes `primePart`/`primeFrase` after each drag to re-anchor the pending slot on the new frontier (else a following click closes at the stale seam — with a phrase covering the necklace end, past the grid). First phrase may back-reach into the previous scene (@/domain/frontier.ts); seam-move threshold `max(3, 25% of scene)` and two-productive-scenes escalation (@/domain/seam.ts).
  - Gates: whole story → scenes → (all triaged + ≥1 productive) → segmentação → (+ ≥1 phrase) → mapeamento; all-none-fit locks downstream.
  - Confidence stored as `high`/`medium`/`low` (ENG-356 — o artefato é inglês); tag states `pending`/`tagged`/`none_fit`.
  - Mapeamento scripts: 11 L1 questions, 5 L2 per locked scene (none_fit included), 5 L3 per phrase in productive scenes — exact order and wording (@/domain/mapeamento-scripts.ts); voice answers keyed as `respostas/level{1,2,3}/…/<k>.webm` (@/domain/mapping.ts).

### Things to Know

- **FROZEN LAYER / contract-critical:** any PR touching this folder requires human review; autonomous loop agents must stop and escalate rather than merge. Change requires the golden harness green AND explicit human approval.
- Coverage gate: ≥ 90% line+branch, enforced per-glob in @/vitest.config.ts. Every PRD §11 rule needs an explicit test, including edge thresholds (seam-move, partial bead, back-reach, boundary drag). Snapshot tests do not count as coverage.
- `scene_kind` values are **English** and come from a generated list — never hand-edit; PT-BR labels are display-only and live in the UI.
- Complexity warnings are expected and accepted here: the port mirrors reference logic 1:1, so functions may exceed the lint's warn threshold (see @/eslint.config.js).
- When in doubt about behavior, read the reference's actual code — its behavior wins over any intuition or existing doc. Never "improve" reference behavior; fidelity is the requirement. Concretely: the `1e-9` epsilon and the `+(x).toFixed(6)` time coercion in @/domain/grid.ts, and the "quirks" of @/domain/hash.ts (only the 2 low bytes of sampleRate are mixed, only channel 0 is read, `Math.imul` because the naive `h * prime` product exceeds 2^53 and silently loses bits) are all byte-identity contract — do not fix them.
- Gate quirk mirrored ON PURPOSE (@/domain/gates.ts): `resolveMode`/`setMode` redirect on productive scenes ONLY, never phrases, so the guided flow can reach `mapeamento` with zero locked phrases; the `modeLocks` tab lock is stricter (requires a locked phrase with span). All-none-fit locks downstream because zero productive keeps segmentacao/mapeamento redirecting back to triagem (`allNoneFit` in @/domain/coverage.ts names the same condition). `scene_kind`/confidence are stored only via @/domain/triagem.ts; PT-BR labels in @/domain/scene-kinds.ts (`SK_PT`) are display-only and never enter the stored data.
- Reference quirks in @/domain/scenes.ts are mirrored ON PURPOSE and are behavior contract, not bugs — e.g. every `confirmPart` leaves a fresh unlocked slot primed at the seam (it occupies a `PT#` and thus shapes ID allocation) while `confirmParts` discards leftover unlocked slots (their `PT#`s return to the pool); and entering the parts layer resumes on the LAST unlocked slot (an artifact of the reference's loop).
- Phrase-layer quirks in @/domain/phrases.ts, equally contract: `confirmFrase` does NOT check `pendingStart`, so a half-selection `{b,b}` confirms as a one-bead phrase; `phraseFrontier` with an active scene does NOT clamp at `totalBeads−1` (only the generic branch does); dangling unlocked slots occupy their `P#` (auto-added by `enterScene`/`enterFrasesLayer`/lock); the cursor is asymmetric — locking and `enterScene` resume on the FIRST unlocked slot, `removeFrase` and `enterFrasesLayer` on the LAST; and when a selection crosses BOTH borders, the END side decides neighbor and delta in @/domain/seam.ts.
- Mapping quirks in @/domain/mapping.ts, mirrored ON PURPOSE: `ensureMapping` seeds with `== null`, so an explicitly stored `""` survives re-seeding; it only EXTENDS — re-cutting or dragging a boundary never deletes an answer bucket, so answers outlive their scene/phrase; L3 buckets are seeded for every locked phrase with span + `part_link` even in a NON-productive scene, while `productiveFrases`/`questionSequence` filter by productive scene — the two views intentionally disagree. The script strings in @/domain/mapeamento-scripts.ts are byte-frozen: the colocated test extracts the `var L1_Q/L2_Q/L3_Q` literals from @/docs/reference/index.html at runtime (via `new Function`) and compares byte-for-byte, plus an NFC guard against hand-pasted NFD — never "fix" punctuation, spacing, or the empty/missing `field` quirks.
- The border-crossing decision is effects-as-data: `classifyBorderMove` (@/domain/seam.ts) returns a `BorderOffer` value and mutates nothing; the UI (or the golden replayer) interprets it into `moveBorder`/`reanchorFrase`/back-to-triagem. The `max(3, round(0.25·span))` threshold is exact in IEEE-754 (0.25 is a power of two; ties round toward +∞ by spec), so it is deterministic across engines.
- `warnedEmptyScene` (the once-per-scene empty warning of `confirmFrasesDone`) is a module variable in the reference; here it is an explicit parameter/return — ephemeral UI state that deliberately does NOT enter `SessionState`. Each caller carries it between calls: the replayer in @/tests/golden/registry.ts keeps a replay-local marker, and the future Segmentação station will keep it as local UI state.
- `enterSegmentacao` is only the scene-focus entry block that the reference runs inside `setMode('segmentacao')`; compose it AFTER `setMode` from @/domain/gates.ts. Inside the layer the dependency is one-way: @/domain/phrases.ts imports @/domain/seam.ts (plus gates/triagem/frontier/ids), never the reverse; @/domain/seam.ts imports only @/domain/scene-kinds.ts and state types.
- Deliberate, documented deviations from the reference exist only where the reference would fail silently or crash on inputs it never produces — byte-identity is unaffected. Examples: `buildBeads` validates `beadSec` itself (the reference validated in its UI caller), `spanDur` throws a typed error instead of a `TypeError` on `undefined`, `setAnswer` throws an explicit Error on a missing L2/L3 bucket or a pre-`ensureMapping` write (the reference would `TypeError` — a programming error, not a flow), and `whole.span` is typed non-null because `createSession` — the only constructor — always sets it. Conversely, `beadAtTime` intentionally carries NO guard: it mirrors the reference exactly.
- Phrase-layer quirks in @/domain/phrases.ts, equally contract: `confirmFrase` does NOT check `pendingStart`, so a half-selection `{b,b}` confirms as a one-bead phrase; the phrase frontier with an active scene does NOT clamp at `totalBeads−1` — only the generic branch of `frontier` in @/domain/frontier.ts does — so with the last phrase covering the end of the necklace the frontier lands OUTSIDE the grid; dangling unlocked slots occupy their `P#` (auto-added by `enterScene`/`enterFrasesLayer`/lock); the cursor is asymmetric — locking and `enterScene` resume on the FIRST unlocked slot, `removeFrase` and `enterFrasesLayer` on the LAST; and when a selection crosses BOTH borders, the END side decides neighbor and delta in @/domain/seam.ts.
- The border-crossing decision is effects-as-data: `classifyBorderMove` (@/domain/seam.ts) returns a `BorderOffer` value and mutates nothing; the UI (or the golden replayer) interprets it into `moveBorder`/`reanchorFrase`/back-to-triagem. The `max(3, round(0.25·span))` threshold is exact in IEEE-754 (0.25 is a power of two; ties round toward +∞ by spec), so it is deterministic across engines.
- `warnedEmptyScene` (the once-per-scene empty warning of `confirmFrasesDone`) is a module variable in the reference; here it is an explicit parameter/return — ephemeral UI state that deliberately does NOT enter `SessionState`. Each caller carries it between calls: the replayer in @/tests/golden/registry.ts keeps a replay-local marker, and the future Segmentação station will keep it as local UI state.
- `enterSegmentacao` is only the scene-focus entry block that the reference runs inside `setMode('segmentacao')`; compose it AFTER `setMode` from @/domain/gates.ts. Inside the layer the dependency is one-way, `phrases → frontier → seam → triagem → state` (plus gates/ids/scene-kinds as leaves): @/domain/phrases.ts imports @/domain/frontier.ts and @/domain/seam.ts, never the reverse. This is why `activeScene` lives in @/domain/seam.ts — keeping it in @/domain/phrases.ts would create the cycle `frontier → phrases → frontier` (the depcruise no-cycles rule is an error).
- Deliberate, documented deviations from the reference exist only where the reference would fail silently or crash on inputs it never produces — byte-identity is unaffected. Examples: `buildBeads` validates `beadSec` itself (the reference validated in its UI caller), `spanDur` throws a typed error instead of a `TypeError` on `undefined`, and `whole.span` is typed non-null because `createSession` — the only constructor — always sets it. Conversely, `beadAtTime` intentionally carries NO guard: it mirrors the reference exactly.
- Validation in the reducers never throws: callers branch on discriminated results (`SceneResult`, `ConfirmFraseResult`, `FrasesDoneResult`), and the exact PT-BR strings live as contract-level copy in `SCENE_ERROR_COPY` (@/domain/scenes.ts), `FRASE_ERROR_COPY`/`FRASES_EMPTY_WARNING` (@/domain/phrases.ts) and `BORDER_COPY` (@/domain/seam.ts) — the UI must reuse them, never rephrase. Silent no-ops are also part of the port — e.g. `confirmPart` on a locked/absent slot returns `ok` without changing state, and `clickBead` ignores clicks while `review` is on.
- @/domain/hash.test.ts duplicates the harness's seeded BigInt LCG PCM generator on purpose: domain code **and its tests** may not import from `tests/` (dependency-cruiser rule), so it cannot reuse @/tests/golden/pcm.ts.
- No network calls, no `Date.now`-style ambient IO, no framework types. If a module seems to need one, the design is wrong — push it out to @/adapters behind a port.
