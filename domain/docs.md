# Noridoc: domain

Path: @/domain

### Overview

- The **frozen core**: pure TypeScript, zero imports (no other layer, no npm packages), a 1:1 behavioral port of the v1 reference at @/docs/reference/index.html.
- Modules land in dependency order via the E1 issues, all re-exported through the @/domain/index.ts barrel (which also keeps the `DOMAIN_LAYER` marker). Landed: ENG-214 вЂ” bead grid math (@/domain/grid.ts), FNV-1a `manifest_id` hash (@/domain/hash.ts), lowest-free ID allocators (@/domain/ids.ts); ENG-216 вЂ” session state (@/domain/state.ts), frontier/active anchor (@/domain/frontier.ts), the bead-click selection model (@/domain/selection.ts), and whole-story + sequential scene cutting (@/domain/scenes.ts); ENG-219 вЂ” the generated Ruth scene-kind ontology (@/domain/scene-kinds.ts), the triagem picker mutations + selectors (@/domain/triagem.ts), the coverage derivation (@/domain/coverage.ts), and the mode gates (@/domain/gates.ts); ENG-223 вЂ” the phrase lifecycle (@/domain/phrases.ts) and the seam/border-crossing logic (@/domain/seam.ts); ENG-226 вЂ” the verbatim mapeamento question scripts (@/domain/mapeamento-scripts.ts) and the lazy answer store (@/domain/mapping.ts). With ENG-226 the E1 domain chain is complete.
- Exists as its own layer so the entire rule set compiles and tests with no framework/IO, making byte-identical artifact reproduction provable by the golden harness.

### How it fits into the larger codebase

- Innermost layer of `ui в†’ adapters в†’ contracts в†’ domain`. The zero-import rule is mechanical, not conventional: @/.dependency-cruiser.cjs errors on any import of another layer **and** on any npm package from production code (tests may import vitest).
- Consumed by: @/contracts (DTO mappers over domain types), @/adapters, `ui/organisms` and `ui/state` (domain types/state via props/hooks вЂ” see @/ui/docs.md).
- The golden harness (@/tests/golden) replays scripted decision sets through `domain/` + `contracts/` and byte-diffs the artifacts against goldens generated from the reference вЂ” proving `domain в‰Ў reference` on every PR. This is why the layer is frozen: its behavior _is_ the product contract. The harness replayers in @/tests/golden/registry.ts import through the @/domain/index.ts barrel (grid/hash for the manifest cases; `createSession` and the scene/triagem/phrase/mapping reducers for `replaySessionSteps`).
- Effects are values, never side effects: reducers return new state (plus a `PlayAction` in @/domain/selection.ts and the `BorderOffer` in @/domain/seam.ts); the interpreters вЂ” audio playback, border-offer dialogs вЂ” live in @/adapters and the UI wiring layers, and UI organisms consume state via props/hooks only.

### Core Implementation

- @/domain/grid.ts вЂ” `buildBeads` (grid = `floor(dur/beadSec + 1e-9)` beads plus a partial bead when a remainder > 1e-9 exists; every `startTime`/`endTime` is coerced through `+(x).toFixed(6)` and `endTime` is clamped to the duration), `beadAtTime` (time в†’ clamped bead index, same epsilon), `spanDur` (span duration from bead times; throws on out-of-grid indices вЂ” a `noUncheckedIndexedAccess`-driven guard).
- @/domain/hash.ts вЂ” `PcmLike` (the minimal AudioBuffer-shaped interface: channels, sampleRate, `getChannelData`) + `hashPCM`, the FNV-1a 32-bit `manifest_id` (`fnv1a32:xxxxxxxx`). Byte-mix order is contract: channel count, the 2 low bytes of sampleRate, sample count as 4 bytes LE, strided int16-quantized samples from channel 0 (stride = `max(1, floor(N/100000))`), then `beadSec*1000` as 3 bytes LE.
- @/domain/ids.ts вЂ” `nextPartId` (`PT#`) and `nextPid` (`P#`): lowest free number, scanning ALL entries вЂ” locked or not, because pending slots occupy their ID.
- @/domain/state.ts вЂ” the COMPLETE `SessionState` shape (whole/parts/frases/current/activeSceneId/mapping/selection/pendingStart/mode/review plus the audio facts: duration, beadSec, bead grid, `manifest_id`, filename, slug) and `createSession`, mirroring the reference's `state` object (reference L367вЂ“379) and the reset done by `segment()` (L454вЂ“462). Later issues only ADD functions over this shape вЂ” they never edit the state module. UI/playback fields of the reference (`color`, `beadSize`, `paused`, download flags, map-step scaffolding, `audioBuffer`) are deliberately excluded: they belong to @/ui and @/adapters.
- @/domain/frontier.ts вЂ” `frontier(state, layer)`, the COMPLETE frontier of both layers, and `activeAnchor` (the current unlocked item, if any, with its frontier as anchored start). On the phrase layer with a productive scene in focus it runs the active-scene branch first: last locked phrase end in that scene + 1; with no phrase in the scene, back-reach to the START of the previous locked neighbor (via `prevNeighbor` from @/domain/seam.ts), else the scene's own start вЂ” and this branch returns BEFORE the clamp. Only when no productive scene is in focus does it fall to the generic branch (max locked end + 1, capped at the last bead; the phrase layer floors at `whole.span.s`). Housing the full branch here (rather than only in @/domain/phrases.ts) is what makes `activeAnchor().start` вЂ” and therefore the `clickBead` clamp in @/domain/selection.ts вЂ” honor back-reach; the earlier split left clicks floored at 0 before the first phrase.
- @/domain/selection.ts вЂ” `clickBead` (reference pointerdown, L561вЂ“583): first click sets `pendingStart` + unit selection, second click normalizes the range, later clicks nudge the nearest edge (tie moves the start), all clamped to [frontier, whole end]. Returns `{state, play}` where `play` is a discriminated `PlayAction` (`single-bead` / `range` / `edge` with a `max(1, round(1/beadSec))`-bead window per playEdge L599вЂ“605 / `transport`). With no active anchor the click is transport (PRD В§8.2) and state is untouched.
- @/domain/scenes.ts вЂ” whole-story confirm and the sequential scene-cutting reducers (`addPart`/`primePart`/`confirmPart`/`confirmParts`, reference L685вЂ“767 and `enterLayer('parts')` L930вЂ“935). All reopen (whole L677вЂ“680 and scene L726вЂ“731) was removed in ENG-342 вЂ” a locked scene is adjusted by dragging its boundary (`dragSceneBoundary`, @/domain/seam.ts), and the story is un-confirmed via the Cortar back action (`setMode('escuta')` + `whole.confirmed=false`), not a reducer here. Validation failures return a typed `Result` (`{ok:false, error:{code, message}}`) whose PT-BR messages are contractual copy exported as `SCENE_ERROR_COPY` вЂ” the UI must reuse them, never rephrase. `confirmParts` prunes unlocked slots and transitions `mode` to `triagem` directly (the mode gates proper live in @/domain/gates.ts).
- @/domain/scene-kinds.ts вЂ” the generated 27-kind Ruth ontology (`SCENE_KINDS`, tier + verbatim order from `_spec/scene-kind-palette.json`, pin 5314907 вЂ” never hand-edit). Each kind's English `value` IS the Compilador contract and is exported as-is; `SK_PT` holds PT-BR display labels (display-only, never stored) and `skEnShort`/`skShort` produce the short labels the report `.md` bytes depend on (ENG-233). `T_TARGET` is the per-tier coverage target (ALTA=1, comum=3).
- @/domain/triagem.ts вЂ” the picker: mutations `tagScene` (sets `tag_state='tagged'` + `scene_kind` + confidence) and `markNoneFit` (`tag_state='none_fit'`, nulls kind/confidence), plus selectors `lockedParts` (locked && span) and `productiveScenes` (locked && span && tagged, in parts order). Pure immutable maps keyed by `part_id`; no throw.
- @/domain/coverage.ts вЂ” `computeCoverage`, a pure port of the reference's `renderCoverage` calc with its DOM side effects and its `updateModeLocks`/`updateTriagemDone` calls stripped (those are now pure in @/domain/gates.ts). Firm = alta|mГ©dia, hesitant = baixa; per-kind status covered/partial/open; `candidateAbsence` = tier ALTA && firm===0 (the reference's open-rara predicate вЂ” ignores hesitant); `allNoneFit` lockout = every locked scene triaged && zero productive. Also exposes additive aggregate derivations (total, rare totals, noneFit, productive, triaged) for the ENG-225 coverage drawer.
- @/domain/gates.ts вЂ” the mode gates + triagem hard gate as pure functions: `triagemDone` (enabled iff every locked scene is non-pending && в‰Ґ1 productive; its PT-BR helper copy is contract-level, verbatim incl. curly quotes/em dash), `modeLocks` (escuta always; triagemв‡”`partsConfirmed`; segmentacaoв‡”productive>0; mapeamentoв‡”productive>0 && в‰Ґ1 locked phrase with span), `resolveMode` (the setMode redirect вЂ” segmentacao/mapeamento with zero productive fall back to triagem; it checks ONLY productive, not phrases), and `setMode` (applies `resolveMode`, always drops `review`, and sets `partsConfirmed=true` when entering segmentacao with locked parts вЂ” the reference L1005 side effect).
- @/domain/seam.ts вЂ” the seam between scenes, all pure data: `prevNeighbor`/`nextNeighbor` (nearest LOCKED neighbors вЂ” none_fit counts; they return `AnchoredPart`, span proven in the type), `sceneHasFrases`, `classifyBorderMove` (the border-crossing offer as a `BorderOffer` value вЂ” kinds `simple`/`escalation`/`two-productive`, threshold `max(3, round(0.25В·span))`, `consumed` = the selection swallows the whole neighbor, `canMove`, contractual PT-BR copy in `BORDER_COPY`; throws on a productive scene without span/kind вЂ” caller-invariant violation, not flow), `slideSeam` (the scene grows, the immediate locked neighbor shrinks; immutable, untouched parts keep identity), and the segmentaГ§ГЈo render window `windowMargin`/`sceneWindow` (`max(3, round(2/beadSec))`).
- @/domain/phrases.ts вЂ” the phrase lifecycle over the productive-scene view of @/domain/triagem.ts: `activeScene`/`sceneIndexOf` (focus resolves by `activeSceneId`, else first productive), `phraseFrontier` (inside the active scene: last locked phrase end + 1; the FIRST phrase back-reaches to the START of the previous locked neighbor, else the scene start), `addFrase` (lowest free `P#` via @/domain/ids.ts; no-op while an anchor is active), `confirmFrase` (guards in the reference's exact order, returning a discriminated `locked`/`border`/`noop`/`error` union with contractual copy in `FRASE_ERROR_COPY`; a border crossing returns the `BorderOffer` вЂ” the UI decides), `moveBorder` (slide the seam then lock), `reanchorFrase`, `removeFrase`, `dragPhraseBoundary` (drag a locked phrase's edge вЂ” grow it, shrink the immediate neighbor only when they touch; ENG-342 replaced the old reopen/вљ‘), the navigation entries `enterScene`/`enterFrasesLayer`/`enterSegmentacao`, and `confirmFrasesDone` (empty-scene warning once PER SCENE; advances to the next productive scene or asks for mapeamento via `setMode` from @/domain/gates.ts).
- @/domain/mapeamento-scripts.ts вЂ” the verbatim Mapeamento question scripts `L1_Q`/`L2_Q`/`L3_Q` (reference L1030вЂ“1056). The strings ARE the byte-exact contract the ENG-233 report `.md` serializes вЂ” including load-bearing unicode (curly quotes U+201C/U+201D in the `tempo` note, em dash U+2014 inside the `funcao` question and `ausencia` note) and structural quirks: L2 `descrever` has an EMPTY `field` (not absent), L3 items have NO `field` property at all.
- @/domain/mapping.ts вЂ” the answer store over the scripts: `ensureMapping` (lazy create/extend of `state.mapping` вЂ” seeds `""` per question with an `== null` check; L2 buckets for ALL locked scenes via `lockedParts` of @/domain/triagem.ts, none_fit included; L3 buckets for every locked phrase with span + `part_link`, even in a non-productive scene; never deletes a bucket), `setAnswer` (direct assignment, the golden driver's semantics; throws on a missing L2/L3 bucket), `productiveFrases` (scene-major order over productive scenes), `questionSequence` (the flat conversation sequence of PRD В§8.7: all L1 в†’ L2 per locked scene in order в†’ L3 per productive-scene phrase), and `voiceAnswerPath` (`respostas/level{1,2,3}/вЂ¦/<k>.webm`, PRD В§10.4 / decision O5).
- @/domain/seam.ts вЂ” the seam between scenes, all pure data: `activeScene` (the productive scene in focus: resolves by `activeSceneId`, else the first productive from @/domain/triagem.ts вЂ” it lives HERE, not in @/domain/phrases.ts, so @/domain/frontier.ts can import it without a cycle), `prevNeighbor`/`nextNeighbor` (nearest LOCKED neighbors вЂ” none_fit counts; they return `AnchoredPart`, span proven in the type), `sceneHasFrases`, `classifyBorderMove` (the border-crossing offer as a `BorderOffer` value вЂ” kinds `simple`/`escalation`/`two-productive`, threshold `max(3, round(0.25В·span))`, `consumed` = the selection swallows the whole neighbor, `canMove`, contractual PT-BR copy in `BORDER_COPY`; throws on a productive scene without span/kind вЂ” caller-invariant violation, not flow), `slideSeam` (the scene grows, the immediate locked neighbor shrinks; immutable, untouched parts keep identity), and the segmentaГ§ГЈo render window `windowMargin`/`sceneWindow` (`max(3, round(2/beadSec))`).
- @/domain/phrases.ts вЂ” the phrase lifecycle over the productive-scene view of @/domain/triagem.ts: `sceneIndexOf`, plus `activeScene` re-exported from @/domain/seam.ts and `phraseFrontier` as a pure alias of `frontier(state,'frases')` in @/domain/frontier.ts (both kept so the @/domain/index.ts public API stayed stable when the logic moved), `addFrase` (lowest free `P#` via @/domain/ids.ts; no-op while an anchor is active), `confirmFrase` (guards in the reference's exact order, returning a discriminated `locked`/`border`/`noop`/`error` union with contractual copy in `FRASE_ERROR_COPY`; a border crossing returns the `BorderOffer` вЂ” the UI decides), `moveBorder` (slide the seam then lock), `reanchorFrase`, `removeFrase`, `dragPhraseBoundary` (drag a locked phrase's edge вЂ” grow it, shrink the immediate neighbor only when they touch; ENG-342 replaced the old reopen/вљ‘), the navigation entries `enterScene`/`enterFrasesLayer`/`enterSegmentacao`, and `confirmFrasesDone` (empty-scene warning once PER SCENE; advances to the next productive scene or asks for mapeamento via `setMode` from @/domain/gates.ts).
- Each module carries colocated tests (`*.test.ts`). Hash tests pin known vectors computed by an independent Python oracle (the emulation script is quoted in @/domain/hash.test.ts) plus vectors derived from the committed goldens in @/tests/golden/expected, so `domain в‰Ў reference` holds both by construction and by byte-diff.
- Non-negotiable facts this layer implements exactly (details in @/docs/PRD-colar-de-sons-v2.md В§10вЂ“В§11 and @/CLAUDE.md):
  - Bead indices are the universal coordinate; `S#` sequence IDs are reassigned only on export.
  - Frontier: sequential locking (@/domain/scenes.ts, @/domain/phrases.ts); post-lock adjustment is dragging a boundary (`dragSceneBoundary`/`dragPhraseBoundary`, ENG-342 вЂ” grow one span, shrink the immediate neighbor); first phrase may back-reach into the previous scene (@/domain/frontier.ts); seam-move threshold `max(3, 25% of scene)` and two-productive-scenes escalation (@/domain/seam.ts).
  - Gates: whole story в†’ scenes в†’ (all triaged + в‰Ґ1 productive) в†’ segmentaГ§ГЈo в†’ (+ в‰Ґ1 phrase) в†’ mapeamento; all-none-fit locks downstream.
  - Confidence stored as `alta`/`mГ©dia`/`baixa`; tag states `pending`/`tagged`/`none_fit`.
  - Mapeamento scripts: 11 L1 questions, 5 L2 per locked scene (none_fit included), 5 L3 per phrase in productive scenes вЂ” exact order and wording (@/domain/mapeamento-scripts.ts); voice answers keyed as `respostas/level{1,2,3}/вЂ¦/<k>.webm` (@/domain/mapping.ts).

### Things to Know

- **FROZEN LAYER / contract-critical:** any PR touching this folder requires human review; autonomous loop agents must stop and escalate rather than merge. Change requires the golden harness green AND explicit human approval.
- Coverage gate: в‰Ґ 90% line+branch, enforced per-glob in @/vitest.config.ts. Every PRD В§11 rule needs an explicit test, including edge thresholds (seam-move, partial bead, back-reach, boundary drag). Snapshot tests do not count as coverage.
- `scene_kind` values are **English** and come from a generated list вЂ” never hand-edit; PT-BR labels are display-only and live in the UI.
- Complexity warnings are expected and accepted here: the port mirrors reference logic 1:1, so functions may exceed the lint's warn threshold (see @/eslint.config.js).
- When in doubt about behavior, read the reference's actual code вЂ” its behavior wins over any intuition or existing doc. Never "improve" reference behavior; fidelity is the requirement. Concretely: the `1e-9` epsilon and the `+(x).toFixed(6)` time coercion in @/domain/grid.ts, and the "quirks" of @/domain/hash.ts (only the 2 low bytes of sampleRate are mixed, only channel 0 is read, `Math.imul` because the naive `h * prime` product exceeds 2^53 and silently loses bits) are all byte-identity contract вЂ” do not fix them.
- Gate quirk mirrored ON PURPOSE (@/domain/gates.ts): `resolveMode`/`setMode` redirect on productive scenes ONLY, never phrases, so the guided flow can reach `mapeamento` with zero locked phrases; the `modeLocks` tab lock is stricter (requires a locked phrase with span). All-none-fit locks downstream because zero productive keeps segmentacao/mapeamento redirecting back to triagem (`allNoneFit` in @/domain/coverage.ts names the same condition). `scene_kind`/confidence are stored only via @/domain/triagem.ts; PT-BR labels in @/domain/scene-kinds.ts (`SK_PT`) are display-only and never enter the stored data.
- Reference quirks in @/domain/scenes.ts are mirrored ON PURPOSE and are behavior contract, not bugs вЂ” e.g. every `confirmPart` leaves a fresh unlocked slot primed at the seam (it occupies a `PT#` and thus shapes ID allocation) while `confirmParts` discards leftover unlocked slots (their `PT#`s return to the pool); and entering the parts layer resumes on the LAST unlocked slot (an artifact of the reference's loop).
- Phrase-layer quirks in @/domain/phrases.ts, equally contract: `confirmFrase` does NOT check `pendingStart`, so a half-selection `{b,b}` confirms as a one-bead phrase; `phraseFrontier` with an active scene does NOT clamp at `totalBeadsв€’1` (only the generic branch does); dangling unlocked slots occupy their `P#` (auto-added by `enterScene`/`enterFrasesLayer`/lock); the cursor is asymmetric вЂ” locking and `enterScene` resume on the FIRST unlocked slot, `removeFrase` and `enterFrasesLayer` on the LAST; and when a selection crosses BOTH borders, the END side decides neighbor and delta in @/domain/seam.ts.
- Mapping quirks in @/domain/mapping.ts, mirrored ON PURPOSE: `ensureMapping` seeds with `== null`, so an explicitly stored `""` survives re-seeding; it only EXTENDS вЂ” re-cutting or dragging a boundary never deletes an answer bucket, so answers outlive their scene/phrase; L3 buckets are seeded for every locked phrase with span + `part_link` even in a NON-productive scene, while `productiveFrases`/`questionSequence` filter by productive scene вЂ” the two views intentionally disagree. The script strings in @/domain/mapeamento-scripts.ts are byte-frozen: the colocated test extracts the `var L1_Q/L2_Q/L3_Q` literals from @/docs/reference/index.html at runtime (via `new Function`) and compares byte-for-byte, plus an NFC guard against hand-pasted NFD вЂ” never "fix" punctuation, spacing, or the empty/missing `field` quirks.
- The border-crossing decision is effects-as-data: `classifyBorderMove` (@/domain/seam.ts) returns a `BorderOffer` value and mutates nothing; the UI (or the golden replayer) interprets it into `moveBorder`/`reanchorFrase`/back-to-triagem. The `max(3, round(0.25В·span))` threshold is exact in IEEE-754 (0.25 is a power of two; ties round toward +в€ћ by spec), so it is deterministic across engines.
- `warnedEmptyScene` (the once-per-scene empty warning of `confirmFrasesDone`) is a module variable in the reference; here it is an explicit parameter/return вЂ” ephemeral UI state that deliberately does NOT enter `SessionState`. Each caller carries it between calls: the replayer in @/tests/golden/registry.ts keeps a replay-local marker, and the future SegmentaГ§ГЈo station will keep it as local UI state.
- `enterSegmentacao` is only the scene-focus entry block that the reference runs inside `setMode('segmentacao')`; compose it AFTER `setMode` from @/domain/gates.ts. Inside the layer the dependency is one-way: @/domain/phrases.ts imports @/domain/seam.ts (plus gates/triagem/frontier/ids), never the reverse; @/domain/seam.ts imports only @/domain/scene-kinds.ts and state types.
- Deliberate, documented deviations from the reference exist only where the reference would fail silently or crash on inputs it never produces вЂ” byte-identity is unaffected. Examples: `buildBeads` validates `beadSec` itself (the reference validated in its UI caller), `spanDur` throws a typed error instead of a `TypeError` on `undefined`, `setAnswer` throws an explicit Error on a missing L2/L3 bucket or a pre-`ensureMapping` write (the reference would `TypeError` вЂ” a programming error, not a flow), and `whole.span` is typed non-null because `createSession` вЂ” the only constructor вЂ” always sets it. Conversely, `beadAtTime` intentionally carries NO guard: it mirrors the reference exactly.
- Phrase-layer quirks in @/domain/phrases.ts, equally contract: `confirmFrase` does NOT check `pendingStart`, so a half-selection `{b,b}` confirms as a one-bead phrase; the phrase frontier with an active scene does NOT clamp at `totalBeadsв€’1` вЂ” only the generic branch of `frontier` in @/domain/frontier.ts does вЂ” so with the last phrase covering the end of the necklace the frontier lands OUTSIDE the grid; dangling unlocked slots occupy their `P#` (auto-added by `enterScene`/`enterFrasesLayer`/lock); the cursor is asymmetric вЂ” locking and `enterScene` resume on the FIRST unlocked slot, `removeFrase` and `enterFrasesLayer` on the LAST; and when a selection crosses BOTH borders, the END side decides neighbor and delta in @/domain/seam.ts.
- The border-crossing decision is effects-as-data: `classifyBorderMove` (@/domain/seam.ts) returns a `BorderOffer` value and mutates nothing; the UI (or the golden replayer) interprets it into `moveBorder`/`reanchorFrase`/back-to-triagem. The `max(3, round(0.25В·span))` threshold is exact in IEEE-754 (0.25 is a power of two; ties round toward +в€ћ by spec), so it is deterministic across engines.
- `warnedEmptyScene` (the once-per-scene empty warning of `confirmFrasesDone`) is a module variable in the reference; here it is an explicit parameter/return вЂ” ephemeral UI state that deliberately does NOT enter `SessionState`. Each caller carries it between calls: the replayer in @/tests/golden/registry.ts keeps a replay-local marker, and the future SegmentaГ§ГЈo station will keep it as local UI state.
- `enterSegmentacao` is only the scene-focus entry block that the reference runs inside `setMode('segmentacao')`; compose it AFTER `setMode` from @/domain/gates.ts. Inside the layer the dependency is one-way, `phrases в†’ frontier в†’ seam в†’ triagem в†’ state` (plus gates/ids/scene-kinds as leaves): @/domain/phrases.ts imports @/domain/frontier.ts and @/domain/seam.ts, never the reverse. This is why `activeScene` lives in @/domain/seam.ts вЂ” keeping it in @/domain/phrases.ts would create the cycle `frontier в†’ phrases в†’ frontier` (the depcruise no-cycles rule is an error).
- Deliberate, documented deviations from the reference exist only where the reference would fail silently or crash on inputs it never produces вЂ” byte-identity is unaffected. Examples: `buildBeads` validates `beadSec` itself (the reference validated in its UI caller), `spanDur` throws a typed error instead of a `TypeError` on `undefined`, and `whole.span` is typed non-null because `createSession` вЂ” the only constructor вЂ” always sets it. Conversely, `beadAtTime` intentionally carries NO guard: it mirrors the reference exactly.
- Validation in the reducers never throws: callers branch on discriminated results (`SceneResult`, `ConfirmFraseResult`, `FrasesDoneResult`), and the exact PT-BR strings live as contract-level copy in `SCENE_ERROR_COPY` (@/domain/scenes.ts), `FRASE_ERROR_COPY`/`FRASES_EMPTY_WARNING` (@/domain/phrases.ts) and `BORDER_COPY` (@/domain/seam.ts) вЂ” the UI must reuse them, never rephrase. Silent no-ops are also part of the port вЂ” e.g. `confirmPart` on a locked/absent slot returns `ok` without changing state, and `clickBead` ignores clicks while `review` is on.
- @/domain/hash.test.ts duplicates the harness's seeded BigInt LCG PCM generator on purpose: domain code **and its tests** may not import from `tests/` (dependency-cruiser rule), so it cannot reuse @/tests/golden/pcm.ts.
- No network calls, no `Date.now`-style ambient IO, no framework types. If a module seems to need one, the design is wrong вЂ” push it out to @/adapters behind a port.
