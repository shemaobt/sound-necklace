# Noridoc: domain

Path: @/domain

### Overview

- The **frozen core**: pure TypeScript, zero imports (no other layer, no npm packages), a 1:1 behavioral port of the v1 reference at @/docs/reference/index.html.
- Modules land in dependency order via the E1 issues, all re-exported through the @/domain/index.ts barrel (which also keeps the `DOMAIN_LAYER` marker). Landed: ENG-214 — bead grid math (@/domain/grid.ts), FNV-1a `manifest_id` hash (@/domain/hash.ts), lowest-free ID allocators (@/domain/ids.ts); ENG-216 — session state (@/domain/state.ts), frontier/active anchor (@/domain/frontier.ts), the bead-click selection model (@/domain/selection.ts), and whole-story + sequential scene cutting (@/domain/scenes.ts). Remaining: ENG-219 (triagem/gates) → ENG-223 (phrases/seam) → ENG-226 (mapeamento scripts + answer store).
- Exists as its own layer so the entire rule set compiles and tests with no framework/IO, making byte-identical artifact reproduction provable by the golden harness.

### How it fits into the larger codebase

- Innermost layer of `ui → adapters → contracts → domain`. The zero-import rule is mechanical, not conventional: @/.dependency-cruiser.cjs errors on any import of another layer **and** on any npm package from production code (tests may import vitest).
- Consumed by: @/contracts (DTO mappers over domain types), @/adapters, `ui/organisms` and `ui/state` (domain types/state via props/hooks — see @/ui/docs.md).
- The golden harness (@/tests/golden) replays scripted decision sets through `domain/` + `contracts/` and byte-diffs the artifacts against goldens generated from the reference — proving `domain ≡ reference` on every PR. This is why the layer is frozen: its behavior _is_ the product contract. The harness replayers in @/tests/golden/registry.ts import through the @/domain/index.ts barrel (grid/hash for the manifest cases; `createSession` and the scene reducers for `replaySessionSteps`).
- Effects are values, never side effects: reducers return new state (plus a `PlayAction` in @/domain/selection.ts); the interpreter that actually plays audio lives in @/adapters, and UI organisms consume state via props/hooks only.

### Core Implementation

- @/domain/grid.ts — `buildBeads` (grid = `floor(dur/beadSec + 1e-9)` beads plus a partial bead when a remainder > 1e-9 exists; every `startTime`/`endTime` is coerced through `+(x).toFixed(6)` and `endTime` is clamped to the duration), `beadAtTime` (time → clamped bead index, same epsilon), `spanDur` (span duration from bead times; throws on out-of-grid indices — a `noUncheckedIndexedAccess`-driven guard).
- @/domain/hash.ts — `PcmLike` (the minimal AudioBuffer-shaped interface: channels, sampleRate, `getChannelData`) + `hashPCM`, the FNV-1a 32-bit `manifest_id` (`fnv1a32:xxxxxxxx`). Byte-mix order is contract: channel count, the 2 low bytes of sampleRate, sample count as 4 bytes LE, strided int16-quantized samples from channel 0 (stride = `max(1, floor(N/100000))`), then `beadSec*1000` as 3 bytes LE.
- @/domain/ids.ts — `nextPartId` (`PT#`) and `nextPid` (`P#`): lowest free number, scanning ALL entries — locked or not, because pending slots occupy their ID.
- @/domain/state.ts — the COMPLETE `SessionState` shape (whole/parts/frases/current/activeSceneId/mapping/selection/pendingStart/mode/review plus the audio facts: duration, beadSec, bead grid, `manifest_id`, filename, slug) and `createSession`, mirroring the reference's `state` object (reference L367–379) and the reset done by `segment()` (L454–462). Later issues only ADD functions over this shape — they never edit the state module. UI/playback fields of the reference (`color`, `beadSize`, `paused`, download flags, map-step scaffolding, `audioBuffer`) are deliberately excluded: they belong to @/ui and @/adapters.
- @/domain/frontier.ts — `frontier(state, layer)` (the generic locked-frontier branch, reference L411–415: max locked end + 1, capped at the last bead; the phrase layer floors at `whole.span.s`) and `activeAnchor` (L416–423: the current unlocked item, if any, with its frontier as anchored start). The phrase-layer active-scene branch (back-reach) arrives with ENG-223.
- @/domain/selection.ts — `clickBead` (reference pointerdown, L561–583): first click sets `pendingStart` + unit selection, second click normalizes the range, later clicks nudge the nearest edge (tie moves the start), all clamped to [frontier, whole end]. Returns `{state, play}` where `play` is a discriminated `PlayAction` (`single-bead` / `range` / `edge` with a `max(1, round(1/beadSec))`-bead window per playEdge L599–605 / `transport`). With no active anchor the click is transport (PRD §8.2) and state is untouched.
- @/domain/scenes.ts — whole-story confirm/reopen and the sequential scene-cutting reducers (`addPart`/`primePart`/`confirmPart`/`reopenPart`/`confirmParts`, reference L677–767 and `enterLayer('parts')` L930–935). Validation failures return a typed `Result` (`{ok:false, error:{code, message}}`) whose PT-BR messages are contractual copy exported as `SCENE_ERROR_COPY` — the UI must reuse them, never rephrase. `confirmParts` prunes unlocked slots and transitions `mode` to `triagem` directly (mode gates proper are ENG-219).
- Each module carries colocated tests (`*.test.ts`). Hash tests pin known vectors computed by an independent Python oracle (the emulation script is quoted in @/domain/hash.test.ts) plus vectors derived from the committed goldens in @/tests/golden/expected, so `domain ≡ reference` holds both by construction and by byte-diff.
- Planned module areas still to land (recorded in @/docs/architecture.md §2): triagem · coverage · mode gates · scene-kinds (generated list) · phrases · seam · mapeamento scripts (verbatim wording) · mapping answer store.
- Non-negotiable facts this layer implements exactly (details in @/docs/PRD-colar-de-sons-v2.md §10–§11 and @/CLAUDE.md):
  - Bead indices are the universal coordinate; `S#` sequence IDs are reassigned only on export.
  - Frontier: sequential locking; reopening item _i_ unlocks _i_ and everything after (live in @/domain/frontier.ts and @/domain/scenes.ts); first phrase may back-reach into the previous scene; seam-move threshold `max(3, 25% of scene)`; two-productive-scenes escalation (ENG-223).
  - Gates: whole story → scenes → (all triaged + ≥1 productive) → segmentação → (+ ≥1 phrase) → mapeamento; all-none-fit locks downstream.
  - Confidence stored as `alta`/`média`/`baixa`; tag states `pending`/`tagged`/`none_fit`.

### Things to Know

- **FROZEN LAYER / contract-critical:** any PR touching this folder requires human review; autonomous loop agents must stop and escalate rather than merge. Change requires the golden harness green AND explicit human approval.
- Coverage gate: ≥ 90% line+branch, enforced per-glob in @/vitest.config.ts. Every PRD §11 rule needs an explicit test, including edge thresholds (seam-move, partial bead, back-reach, reopen cascade). Snapshot tests do not count as coverage.
- `scene_kind` values are **English** and come from a generated list — never hand-edit; PT-BR labels are display-only and live in the UI.
- Complexity warnings are expected and accepted here: the port mirrors reference logic 1:1, so functions may exceed the lint's warn threshold (see @/eslint.config.js).
- When in doubt about behavior, read the reference's actual code — its behavior wins over any intuition or existing doc. Never "improve" reference behavior; fidelity is the requirement. Concretely: the `1e-9` epsilon and the `+(x).toFixed(6)` time coercion in @/domain/grid.ts, and the "quirks" of @/domain/hash.ts (only the 2 low bytes of sampleRate are mixed, only channel 0 is read, `Math.imul` because the naive `h * prime` product exceeds 2^53 and silently loses bits) are all byte-identity contract — do not fix them.
- Reference quirks in @/domain/scenes.ts are mirrored ON PURPOSE and are behavior contract, not bugs — e.g. `reopenWhole` does NOT clear the selection; every `confirmPart` leaves a fresh unlocked slot primed at the seam (it occupies a `PT#` and thus shapes ID allocation) while `confirmParts` discards leftover unlocked slots (their `PT#`s return to the pool); and entering the parts layer resumes on the LAST unlocked slot (an artifact of the reference's loop).
- Deliberate, documented deviations from the reference exist only where the reference would fail silently or crash on inputs it never produces — byte-identity is unaffected. Examples: `buildBeads` validates `beadSec` itself (the reference validated in its UI caller), `spanDur` throws a typed error instead of a `TypeError` on `undefined`, `reopenPart` is a no-op on an out-of-range index (the reference would crash), and `whole.span` is typed non-null because `createSession` — the only constructor — always sets it. Conversely, `beadAtTime` intentionally carries NO guard: it mirrors the reference exactly.
- Validation in scene reducers never throws: callers branch on the discriminated `SceneResult`, and the exact PT-BR error strings live in `SCENE_ERROR_COPY` (@/domain/scenes.ts) as contract-level copy. Silent no-ops are also part of the port — e.g. `confirmPart` on a locked/absent slot returns `ok` without changing state, and `clickBead` ignores clicks while `review` is on.
- @/domain/hash.test.ts duplicates the harness's seeded BigInt LCG PCM generator on purpose: domain code **and its tests** may not import from `tests/` (dependency-cruiser rule), so it cannot reuse @/tests/golden/pcm.ts.
- No network calls, no `Date.now`-style ambient IO, no framework types. If a module seems to need one, the design is wrong — push it out to @/adapters behind a port.
