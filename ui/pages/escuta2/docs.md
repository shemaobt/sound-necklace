# Noridoc: escuta2

Path: @/ui/pages/escuta2

### Overview

- The second flow station (ENG-230): "Escuta 2 — o corte de cenas" (PRD v2 §8.4, redesign §6.3), where the listener cuts the confirmed whole story into scenes by tapping only where **each scene ends** — the start of every scene comes pre-anchored at the previous seam by the domain.
- A wiring component: it reads the pure @/domain session through the @/ui/state session store, renders the @/ui/organisms `Necklace` with anchoring active, and dispatches the scene-cut/confirm/reopen domain reducers.
- Cream working stage with exactly one instruction line ("Toque no colar onde **esta cena termina**") and exactly one dominant action ("✓ Confirmar esta cena"), per the listener-facing UI rules (§9.2).

### How it fits into the larger codebase

- **The station after @/ui/pages/escuta1.** Stations self-register through the `import.meta.glob('/ui/pages/*/index.tsx')` registry built in @/ui/app/registries.ts, so the default export in `index.tsx` is mandatory — it is the registry value, keyed by the directory name `escuta2`. @/ui/app/App.tsx's `KEY_TO_MODE` maps `escuta2` to the domain `escuta` mode; the app shell renders this station (rather than escuta1) when the session mode is `escuta` **and** `whole.confirmed` is true — the split lives in @/ui/app/stepper-model.ts `currentIndex`.
- **Wiring layer.** Per @/.dependency-cruiser.cjs, @/ui/pages may import adapters, @/domain, @/ui/state, @/ui/organisms, @/ui/molecules, @/ui/atoms and @/ui/tokens — the only place besides @/ui/app and templates allowed to reach adapters.
- Reads session state via `useSessionStore` and writes only through `sessionStore.apply` (see @/ui/state/docs.md), so the store's editability gates (online/review/lock) can silently pause a cut/confirm/reopen without losing in-memory state.
- Advancing dispatches `confirmParts` (@/domain/scenes.ts), which drops the trailing unlocked scene, sets `partsConfirmed`, and moves the guided flow to Triagem (`mode='triagem'`) — so this station is the hand-off from raw listening into scene classification.
- The itinerant `Player` (@/ui/app/player-slot.tsx, @/adapters/audio) is injected by prop, not constructed here — see the audio seam below.

### Core Implementation

- **`Escuta2({ player })`** (@/ui/pages/escuta2/index.tsx): subscribes `session` from the store; renders `null` until a session exists. Local React state holds only the `playbackHead` (mirrored from the player via `player.onHead`) and the last domain error string.
- **Header** (realigned to the Protótipo): the instruction sits under a `title` heading ("Corte a história em cenas", @/ui/i18n `escuta2.title`); the necklace uses the shared `SIZE_L` preset (26px beads, same as @/ui/pages/escuta1). There are **no play buttons on this station** — sound comes from the necklace beads (owner decision, ENG-291/PR #76): no pending-selection play, no per-chip `▶`.
- **The single instruction line adapts to what is possible** (§9.2 allows exactly one): before any scene is locked it explains the seam (`escuta2.instructionPost`, "O começo já está costurado."); once ≥1 scene is locked it signals the replay affordance instead (`escuta2.instructionReplay`, "Toque numa cena pronta para reouvir." — verbatim from the "Ouvir no colar" study, which swaps the second sentence rather than adding a third). The review moment's headline carries the same signal (`escuta2.reviewHeadline`): with every bead locked, replay is the _only_ thing the necklace still does, and this is precisely where the facilitator checks the cut before "Continuar →" — it was the affordance's blindest spot. The hint is never shown before there is a locked scene to replay: an affordance line for something that does not exist yet is noise, not signal.
- **Anchoring necklace** — unlike escuta1's transport-only colar, the necklace runs with anchoring active. Structural props (`segments` = locked parts as `{span, tint}`, `lockedEndBeads` = square end beads) are `useMemo`'d on `session.parts` so per-frame `playbackHead` updates never recompute them, preserving the necklace's no-rerender imperative lighting.
- **Click model — locked scenes are heard, free beads are cut (ENG-293).** `onBeadPointerDown` asks `lockedSceneAt(parts, bead)` **first**: if the bead belongs to an already-locked scene, the station plays that whole scene (`player.toggle(part_id, s, e)`) and returns — the cut in progress is untouched. Otherwise it delegates to the pure domain reducer `clickBead` (@/domain/selection.ts): read the freshest session via `sessionStore.getState()`, compute `{state, play}`, apply through `sessionStore.apply`, then play the returned `PlayAction` via `playActionOn`. Edge hover calls `player.playEdge` directly. `onHeadTap` (ENG-297) honours redesign §127's "the glowing head pauses" **whatever is playing**, by asking about the key first: `play`/`playEdge` are keyless (`playRange` → `stopPlayback` clears `key`, adapters/audio/player.ts:89/110), so a keyless playback gets `stop()` — nothing to resume — while a scene playing under its own key routes to `playLockedSceneAt` and pauses retrievably. Without that split, a head landing inside a locked scene during an _edge preview_ (window = `max(1, round(1/beadSec))` = 4 beads a side, so it reaches back into the previous scene) would re-toggle a key that isn't current and **restart the scene** for someone who tapped to stop; and the head over a free bead during a selection playback was a dead click. This mirrors `_headTapPause` in the "Ouvir no colar" study (`:697`), which is likewise checked before deciding what to play.
  - **Why the interception lives here and not in `clickBead`:** the reducer is a 1:1 port of the v1 reference, which clamps every click to `[frontier, end of story]` — a tap inside a locked scene is clamped up to the seam and _consumes the next scene's pre-anchoring_, closing a degenerate one-bead scene. Teaching the reducer about locked scenes is a frozen-layer change (golden harness is the judge); the station asks the question before delegating instead. The trade-off is real and deliberate: the click model now has two homes.
  - **Side effect, and it is wanted:** asking before the reducer also means asking before its `state.review` early-return (@/domain/selection.ts:27, `play: null`). So under domain review the necklace still plays locked scenes instead of going silent — which is what redesign §5.3 asks of review ("play-only"). Covered by a browser test; do not "tidy" the interception below `clickBead` or review goes mute again.
- **`lockedSceneAt` / `playActionOn` / `sceneLabel` / `sceneColor`** (@/ui/pages/escuta2/cutting.ts): `lockedSceneAt` is the `_sceneOf` of the "Ouvir no colar" study (redesign §11's chosen `pure`) — the locked scene owning a bead, both borders inclusive, `null` for unlocked or span-less parts (a reopened scene keeps its span and only drops `locked`, so the `locked` filter is what lets it be re-cut instead of swallowing the click); `playActionOn` interprets the reducer's effect-as-value `PlayAction` (single-bead/range → `player.play(s,e)`; edge → `player.playEdge`; transport → the bead itself); `sceneLabel` renders "Cena <ordinal-por-extenso>" (um/dois/… up to 99, then bare "Cena") so the screen stays digit-free; `sceneColor` cycles the terracotta `scenePalette` (@/ui/tokens) by part index.
- **Confirm this scene** — the dominant action (`data-role="primary-action"`, exactly one) dispatches `confirmPart(state, current.index)`: it locks the span, squares the end bead, and auto-opens the next scene pre-anchored at the seam (frontier+1). It renders only while `activeAnchor(session)` exists (always true during escuta2).
- **The review moment (`tiled`, owner decision):** once the locked scenes **tile 0…N-1 with no gap** (`tilesWholeStory`, measured — not inferred from the last bead: a saved return brings locked `parts` straight from JSON with arbitrary spans, and trusting the highest end bead would leave an uncut stretch unreachable and silently discarded by `confirmParts`), the residual anchor the domain still opens (`confirmParts` just discards it, PRD §8.4) is hidden and the station collapses to one headline (`escuta2.reviewHeadline`) plus one action (`review.continue`, shared with @/ui/pages/triagem and @/ui/pages/segmentacao) that dispatches the same `confirmParts`. Partial coverage keeps the PRD's own instruction/confirm-scene/"Confirmar as cenas →" flow untouched — the PRD's CTAs are never removed, only superseded once nothing is left to cut. This is also the moment the replay affordance matters most: every bead now belongs to a locked scene, so the whole necklace reads back the cut before "Continuar →".
- **Confirmed scenes** render as `ScenePhraseChip` (swatch · label · Reabrir) in a wrapping row — a named `role="group"`, never a control; `Reabrir` stops the player (the scene being heard is precisely the one about to stop existing — ENG-297) and dispatches `reopenPart(state, i)`, which unlocks scene _i_ and everything after it (frontier-integrity cascade), reflected in the chip list.

### Things to Know

- **"Confirmar as cenas →"** (dark button) appears only when ≥1 scene is locked AND the story is not yet fully tiled by scenes (`!tiled`); it dispatches `confirmParts` and advances to Triagem, discarding the trailing unlocked scene. Once the story IS fully tiled, this button and the "✓ Confirmar esta cena" both hide behind the review headline — see Core Implementation.
- **"← Voltar"** returns to Escuta 1 preserving the cut scenes: it composes `setMode({ ...s, whole: { ...s.whole, confirmed: false } }, 'escuta')` — a faithful port of the reference's `cenasBack` handler (whole.confirmed=false; setMode('escuta')), distinct from escuta1's `reopenWhole`.
- The two confirm-scene error copies come verbatim from the domain contract `SCENE_ERROR_COPY` (`SELECTION_INCOMPLETE` and `SCENE_BEFORE_FRONTIER`, the latter carrying the frontier bead number) — never re-authored in the UI; shown in a `role="alert"` line. Being domain copy, they stay **PT-BR under an EN UI**, unlike the station's own instruction/button copy (@/ui/i18n).
- **The audio seam is the extension point.** `player` defaults to `null` (same seam as escuta1); with no player the click model computes state but skips playback, and no engine is constructed here. Extend audio through this prop.
- **Grid alignment invariant:** the injected player's grid (`beadSec` / decoded duration) must match the session grid (`totalBeads` / `beadSec`) threaded to the necklace, or bead-to-time mapping drifts.
- **Digit-free listener rule (§9.2):** scene chips use spelled ordinals from cutting.ts, not numerals; the necklace's internal `data-idx` bead numbers stay data attributes (not textContent/aria/title) so they don't violate the guard.
- Chip-entry motion (@/ui/pages/escuta2/escuta2.css) is guarded by `@media (prefers-reduced-motion: no-preference)`; bead lighting motion lives in the atoms, already guarded there.
- Tests split by suffix: `cutting.test.ts` and `escuta2.test.tsx` run in jsdom; `escuta2.browser.test.tsx` runs in real Chromium for the coordinate click model against real necklace geometry (jsdom has zeroed layout).
