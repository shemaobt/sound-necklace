# Noridoc: cut

Path: @/ui/pages/cut

### Overview

- The second flow station (ENG-230): "Escuta 2 — o corte de cenas" (PRD v2 §8.4, redesign §6.3), where the listener cuts the confirmed whole story into scenes by tapping only where **each scene ends** — the start of every scene comes pre-anchored at the previous seam by the domain.
- A wiring component: it reads the pure @/domain session through the @/ui/state session store, renders the @/ui/organisms `Necklace` with anchoring active, and dispatches the scene-cut/confirm/boundary-drag domain reducers.
- Cream working stage with exactly one instruction line ("Toque no colar onde **esta cena termina**") and exactly one dominant action ("✓ Confirmar esta cena"), per the listener-facing UI rules (§9.2).

### How it fits into the larger codebase

- **The station after @/ui/pages/listen.** Stations self-register through the `import.meta.glob('/ui/pages/*/index.tsx')` registry built in @/ui/app/registries.ts, so the default export in `index.tsx` is mandatory — it is the registry value, keyed by the directory name `cut`. @/ui/app/App.tsx's `KEY_TO_MODE` maps `cut` to the domain `escuta` mode; the app shell renders this station (rather than listen) when the session mode is `escuta` **and** `whole.confirmed` is true — the split lives in @/ui/app/stepper-model.ts `currentIndex`.
- **Wiring layer.** Per @/.dependency-cruiser.cjs, @/ui/pages may import adapters, @/domain, @/ui/state, @/ui/organisms, @/ui/molecules, @/ui/atoms and @/ui/tokens — the only place besides @/ui/app and templates allowed to reach adapters.
- Reads session state via `useSessionStore` and writes only through `sessionStore.apply` (see @/ui/state/docs.md), so the store's editability gates (online/review/lock) can silently pause a cut/confirm/boundary-drag without losing in-memory state.
- Advancing dispatches `confirmParts` (@/domain/scenes.ts), which drops the trailing unlocked scene, sets `partsConfirmed`, and moves the guided flow to Triage (`mode='triage'`) — so this station is the hand-off from raw listening into scene classification.
- The itinerant `Player` (@/ui/app/player-slot.tsx, @/adapters/audio) is injected by prop, not constructed here — see the audio seam below.

### Core Implementation

- **`Cut({ player })`** (@/ui/pages/cut/index.tsx): subscribes `session` from the store; renders `null` until a session exists. Local React state holds only the `playbackHead` (mirrored from the player via `player.onHead`) and the last domain error string.
- **Header** (realigned to the Protótipo): the instruction sits under a `title` heading ("Corte a história em cenas", @/ui/i18n `cut.title`); the necklace uses the shared `SIZE_L` preset (26px beads, same as @/ui/pages/listen). There are **no play buttons on this station** — sound comes from the necklace beads (owner decision, ENG-291/PR #76): no pending-selection play, no per-chip `▶`.
- **The single instruction line adapts to what is possible** (§9.2 allows exactly one): before any scene is locked it explains the seam (`cut.instructionPost`, "O começo já está costurado."); once ≥1 scene is locked it signals the replay affordance instead (`cut.instructionReplay`, "Toque numa cena pronta para reouvir." — verbatim from the "Ouvir no colar" study, which swaps the second sentence rather than adding a third). The review moment's headline carries the same signal (`cut.reviewHeadline`): with every bead locked, replay is the _only_ thing the necklace still does, and this is precisely where the facilitator checks the cut before "Continuar →" — it was the affordance's blindest spot. The hint is never shown before there is a locked scene to replay: an affordance line for something that does not exist yet is noise, not signal.
- **Anchoring necklace** — unlike listen's transport-only colar, the necklace runs with anchoring active. Structural props (`segments` = locked parts as `{span, tint}`, `lockedEndBeads` = square end beads) are `useMemo`'d on `session.parts` so per-frame `playbackHead` updates never recompute them, preserving the necklace's no-rerender imperative lighting.
- **Click model — locked scenes are heard, free beads are cut (ENG-293; full rules @/docs/segmentation-rules.md).** `onBeadPointerDown` asks `lockedItemAt(parts, bead)` **first**: if the bead belongs to an already-**confirmed** scene, the station plays **FROM the tapped bead** to the scene end (`player.toggle('<part_id>:<bead>', bead, span.e)`, rule 4) and returns — the cut in progress is untouched. Per-bead key: a **different** bead jumps there; the **same** bead pauses/resumes. Otherwise it delegates to `clickBead` (@/domain/selection.ts) and plays via `playClick(player, play, parentEnd, head)` (@/ui/pages/cut/cutting.ts, `parentEnd = whole.span.e`): while **defining**, clicking the **start** (frontier) → `listen` (play from the start to the story end); clicking **beyond** → `set-end` (the domain sets `selection = {frontier, bead}`) and then, from the runtime playhead, **stop if the head already reached the bead, else keep playing** (do not interrupt). The start is never settable (rule 7). Dragging the boundary plays a preview via `playEditWindow` (~4 beads before the limit to ~3 after, rule 5). `onHeadTap` (ENG-297): the `listen`/`set-end`/transport playbacks are keyless (`player.play`) so the glowing head `stop()`s them; a confirmed-scene playback carries a key (`player.toggle`) and is paused/resumed in place by re-toggling the **active** key.
  - **Why the interception lives here and not in `clickBead`:** the reducer is a 1:1 port of the v1 reference, which clamps every click to `[frontier, end of story]` — a tap inside a locked scene is clamped up to the seam and _consumes the next scene's pre-anchoring_, closing a degenerate one-bead scene. Teaching the reducer about locked scenes is a frozen-layer change (golden harness is the judge); the station asks the question before delegating instead. The trade-off is real and deliberate: the click model now has two homes.
  - **Side effect, and it is wanted:** asking before the reducer also means asking before its `state.review` early-return (@/domain/selection.ts:27, `play: null`). So under domain review the necklace still plays locked scenes instead of going silent — which is what redesign §5.3 asks of review ("play-only"). Covered by a browser test; do not "tidy" the interception below `clickBead` or review goes mute again.
- **`lockedItemAt` / `playActionOn` / `rankLockedScenes` / `sceneOrdinal` / `sceneColor` / `sceneLabel`** (@/ui/pages/cut/cutting.ts): `lockedItemAt` is the `_sceneOf` of the "Ouvir no colar" study (redesign §11's chosen `pure`) — the locked scene owning a bead, both borders inclusive, `null` for unlocked or span-less parts (a reopened scene keeps its span and only drops `locked`, so the `locked` filter is what lets it be re-cut instead of swallowing the click); `playActionOn` interprets the reducer's effect-as-value `PlayAction` (single-bead/range → `player.play(s,e)`; edge → `player.playEdge`; transport → the bead itself); `rankLockedScenes` sorts the locked scenes by starting bead and assigns a 0-based `rank` (the number and colour follow necklace position, not array order — a saved return can carry `parts` out of order, ENG-344) while keeping each scene's `arrayIndex`; `sceneOrdinal(rank, lang)` spells the number in the UI language (um/dois/… or one/two/…, empty beyond 99) and the "Cena"/"Scene" prefix comes from the i18n key `cut.sceneLabel`, so the label is digit-free and follows the PT/EN toggle; `sceneColor` cycles the terracotta `scenePalette` (@/ui/tokens) by rank. `sceneLabel` is the legacy PT-only "Cena N" builder kept only for the phrases station (`phraseLabel` and its parent-scene header) until ENG-343 migrates it.
- **Confirm this scene** — the dominant action (`data-role="primary-action"`, exactly one) dispatches `confirmPart(state, current.index)`: it locks the span, squares the end bead, and auto-opens the next scene pre-anchored at the seam (frontier+1). It renders only while `activeAnchor(session)` exists (always true during cut).
- **The review moment (`tiled`, owner decision):** once the locked scenes **tile 0…N-1 with no gap** (`tilesWholeStory`, measured — not inferred from the last bead: a saved return brings locked `parts` straight from JSON with arbitrary spans, and trusting the highest end bead would leave an uncut stretch unreachable and silently discarded by `confirmParts`), the residual anchor the domain still opens (`confirmParts` just discards it, PRD §8.4) is hidden and the station collapses to one headline (`cut.reviewHeadline`) plus one action (`review.continue`, shared with @/ui/pages/triage and @/ui/pages/phrases) that dispatches the same `confirmParts`. Partial coverage keeps the PRD's own instruction/confirm-scene/"Confirmar as cenas →" flow untouched — the PRD's CTAs are never removed, only superseded once nothing is left to cut. This is also the moment the replay affordance matters most: every bead now belongs to a locked scene, so the whole necklace reads back the cut before "Continuar →".
- **Confirmed scenes** render as `ScenePhraseChip` (swatch · label · Remover) in a wrapping row, ordered by necklace position (`rankLockedScenes`) — a named `role="group"`. Remover composes `absorbNextScene(removePart(...), removed.span.s)` — `removePart` is pure/reference-faithful, and the **next** locked scene absorbs the freed gap (stretches its start back; #3, owner decision), mirroring the phrase chip. Reabrir was removed in ENG-342: a locked scene is adjusted by dragging its END on the necklace (`dragSceneBoundary`, via `dragHandles`/`onDragBoundary` — now **Pac-Man/tiled, identical to the phrase**: the next scene follows the boundary in both directions (its start becomes `newEnd+1`), so shrinking never leaves a gap — the next grows to fill — and growing pushes it; the last scene grows/shrinks freely to the story end). Only the END has a handle — the start is the seam. Every drag is composed with `primePart` so the pending scene re-anchors on the new frontier. Full rules: @/docs/segmentation-rules.md.

### Things to Know

- **"Confirmar as cenas →"** (dark button) appears only when ≥1 scene is locked AND the story is not yet fully tiled by scenes (`!tiled`); it dispatches `confirmParts` and advances to Triage, discarding the trailing unlocked scene. Once the story IS fully tiled, this button and the "✓ Confirmar esta cena" both hide behind the review headline — see Core Implementation.
- **"← Voltar"** returns to Escuta 1 preserving the cut scenes: it composes `setMode({ ...s, whole: { ...s.whole, confirmed: false } }, 'escuta')` — a faithful port of the reference's `cenasBack` handler (whole.confirmed=false; setMode('escuta')). Since ENG-342 removed `reopenWhole`, this is the only way to un-confirm the story.
- The two confirm-scene error copies come verbatim from the domain contract `SCENE_ERROR_COPY` (`SELECTION_INCOMPLETE` and `SCENE_BEFORE_FRONTIER`, the latter carrying the frontier bead number) — never re-authored in the UI; shown in a `role="alert"` line. Being domain copy, they stay **PT-BR under an EN UI**, unlike the station's own instruction/button copy (@/ui/i18n).
- **The audio seam is the extension point.** `player` defaults to `null` (same seam as listen); with no player the click model computes state but skips playback, and no engine is constructed here. Extend audio through this prop.
- **Grid alignment invariant:** the injected player's grid (`beadSec` / decoded duration) must match the session grid (`totalBeads` / `beadSec`) threaded to the necklace, or bead-to-time mapping drifts.
- **Digit-free listener rule (§9.2):** scene chips use spelled ordinals from cutting.ts, not numerals; the necklace's internal `data-idx` bead numbers stay data attributes (not textContent/aria/title) so they don't violate the guard.
- Chip-entry motion (@/ui/pages/cut/cut.css) is guarded by `@media (prefers-reduced-motion: no-preference)`; bead lighting motion lives in the atoms, already guarded there.
- Tests split by suffix: `cutting.test.ts` and `cut.test.tsx` run in jsdom; `cut.browser.test.tsx` runs in real Chromium for the coordinate click model against real necklace geometry (jsdom has zeroed layout).
