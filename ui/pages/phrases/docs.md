### How it fits into the larger codebase

- **The station after Triage.** Stations self-register through the `import.meta.glob('/ui/pages/*/index.tsx')` registry built in @/ui/app/registries.ts, so the default export in `index.tsx` is mandatory вЂ” it is the registry value, keyed by the directory name `phrases`. @/ui/app/App.tsx's `KEY_TO_MODE` maps `phrases` to the domain `phrases` mode; the app shell renders this station when the session mode is `phrases`.
- **Wiring layer.** Per @/.dependency-cruiser.cjs, @/ui/pages may import @/domain, @/ui/state, @/ui/organisms, @/ui/molecules, @/ui/atoms, @/ui/tokens and adapters. - Reads session state via `useSessionStore` and writes only through `sessionStore.apply` (see @/ui/state/docs.md), so the store's editability gates (online/review/lock) can silently pause an anchor/move/boundary-drag without losing in-memory state.

- The itinerant `Player` (@/ui/app/player-slot.tsx, @/adapters/audio) is injected by prop, not constructed here вЂ” see the audio seam below.

### Core Implementation

- **`Phrases({ player })`** (@/ui/pages/phrases/index.tsx): subscribes `session` from the store; renders `null` until a session with an active productive scene exists. A single `useMemo` keyed on `session` derives the active scene (`activeScene`), its span, the scene's locked phrases (with their global index), and the necklace's `segments`/`lockedEndBeads` вЂ” one memo so per-frame `playbackHead` updates never recompute the field (the necklace's no-rerender imperative lighting).
- **Header + framed stage** (realigned to the ProtГіtipo): the "Cena N В· tipo" title now carries a small scene-tinted swatch (`sceneColor`, reused from @/ui/pages/cut/cutting.ts) next to the text; the whole necklace stage sits inside a dashed 26px-radius frame (`phrases.css`), and the necklace uses the `SIZE_SEG` preset (30px beads вЂ” the largest of the three presets, since this station is where phrase precision matters most).
- **Windowed necklace** вЂ” the `window` prop is the active scene span; the organism draws the dashed awake band and dims/omits beads outside scene В± `max(3, round(2/beadSec))` (the margin lives in `resolveWindow`). Locked phrases tint the cord via `phraseColor` (cycles `phrasePalette`); their end beads render square.
- **Click model вЂ” locked phrases are heard, free beads are cut (ENG-296).** `onBeadPointerDown` asks `lockedItemAt(scenePhrases, bead)` **first** (the generic `_sceneOf`/`_phraseOf` of the "Ouvir no colar" study, shared with @/ui/pages/cut/cutting.ts вЂ” the rule ignores the id, so scene and phrase use the same one): a bead inside an already-locked phrase plays that whole phrase (`player.toggle(prop_id, s, e)`) and returns, leaving the cut in progress alone. Only phrases of the **active scene** count вЂ” they are the only ones inside the necklace's window. Otherwise it delegates to the pure domain reducer `clickBead` (@/domain/selection.ts): read the freshest session via `sessionStore.getState()`, apply the returned state, then play the `PlayAction` via `playActionOn`. Edge hover calls `player.playEdge`.
  - **Why here and not in `clickBead`:** same reason as cut (see its docs.md) вЂ” the reducer is a frozen 1:1 port that clamps the click to the frontier and would consume the next phrase's pre-anchoring. The click model deliberately has two homes; a third station would be the signal to move the rule into the domain instead.
  - **`onHeadTap` (ENG-297's guard, needed here too):** this station has `onEdgeHover` в†’ `playEdge`, which is **keyless** (`playRange` clears `key`). Without asking the key first, the glowing bead during an edge preview would re-toggle a foreign key and **restart** the phrase for someone tapping to stop. Keyless в†’ `stop()`; own key в†’ pause.
- **Confirm this phrase** вЂ” the dominant action (`data-role="primary-action"`) dispatches `confirmFrase(state, current.index)`, whose result is a discriminated union: `error` (shows the domain copy), `border` (opens the seam modal with the offer), `locked` (applies the new state), `noop`. It renders while `activeAnchor(session)` exists (always true while phrasing a scene).
- **The review moment (`covered`, owner decision):** once the active scene's locked phrases cover it end to end (the highest locked phrase's end bead equals the scene span's end), the station collapses to one headline (`phrases.reviewHeadline`) plus one action (`review.continue`, shared with @/ui/pages/cut and @/ui/pages/triage) that dispatches the same `confirmFrasesDone` вЂ” so a fully-covered scene never shows the empty-scene warning, since coverage already proves at least one phrase exists. A scene with sparse phrases (gaps still open) keeps the PRD's instruction/confirm-phrase/"Pronto com esta cena в†’"/"JГЎ segmentei todas as cenas в†’" flow untouched, because sparse phrases are a deliberately allowed outcome of the frozen domain вЂ” the station never forces full coverage.
- **Seam modal** (@/ui/organisms `SeamModal`): fed the domain `BorderOffer` plus the scene and the immediate neighbor of the crossing side (`nextNeighbor`/`prevNeighbor`, tinted by `sceneColor`). `Mover a borda atГ© aqui` в†’ `moveBorder(state, offer)` (slides the shared seam, then locks); `Reancorar dentro da cena` в†’ `reanchorFrase` (clears the selection); `Voltar Г  Triage` в†’ `setMode('triage')`. Move runs against the same store state that produced the offer (unchanged while the modal is open).
- **Locked phrases** render as `ScenePhraseChip` (swatch В· "Frase N" В· Remover вЂ” no в–¶ since ENG-291: the beads play the phrase), dispatching `removeFrase` by the phrase's global index. Reabrir/вљ‘ revisar were removed in ENG-342 вЂ” post-lock boundary adjustment is now dragging the phrase edge on the necklace (`dragPhraseBoundary`, via the necklace's `dragHandles`/`onDragBoundary`, composed with `primeFrase` so the pending phrase re-anchors on the new frontier вЂ” else, with the prior phrase covering the necklace end, the frontier sits past the grid and the next click closes beyond the story). `phraseLabel` reuses cut's tested cardinal ("Cena N" в†’ "Frase N") so the screen stays digit-free.

### Things to Know

- **Primary button** reads "Pronto com esta cena в†’", or "JГЎ segmentei todas as cenas в†’" on the last productive scene (`sceneIndexOf` vs `productiveScenes.length`) вЂ” only while the scene isn't `covered`; once covered, both collapse into the single `review.continue` button (see Core Implementation), which still dispatches `confirmFrasesDone`.
- **Empty-scene soft warning:** leaving a scene with zero locked phrases surfaces "Esta cena ficou sem frases. Clique de novo para seguir mesmo assim." once (the `warnedEmptyScene` marker is UI-local state); a second click proceeds. Reference-faithful.
- **"в†ђ Voltar"** goes to the previous productive scene (`enterScene`), or to Triage from the first (`setMode('triage')`) вЂ” a port of the reference's `frasesBack`.
- All validation copy comes verbatim from `FRASE_ERROR_COPY` / `FRASES_EMPTY_WARNING` (@/domain/phrases.ts) вЂ” never re-authored in the UI; shown in a `role="alert"` line. The only digit any listener sees is the facilitator bead number interpolated into the "phrase starts before bead X" message (В§9.2 carve-out). Being domain copy, all of it stays **PT-BR under an EN UI**; the station's own copy and the seam modal translate (@/ui/i18n).
- **The audio seam is the extension point.** `player` defaults to `null`; with no player the click model computes state but skips playback, and no engine is constructed here. Extend audio through this prop.
- Chip-entry motion (@/ui/pages/phrases/phrases.css) is guarded by `@media (prefers-reduced-motion: no-preference)`; bead lighting motion lives in the atoms, already guarded there.
- Tests split by suffix: `wiring.test.ts` and `phrases.test.tsx` run in jsdom; `phrases.browser.test.tsx` runs in real Chromium for the coordinate click model against real windowed necklace geometry (jsdom has zeroed layout).
